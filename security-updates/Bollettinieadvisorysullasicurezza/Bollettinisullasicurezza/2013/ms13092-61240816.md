Security Bulletin

Bollettino Microsoft sulla sicurezza MS13-092 - Importante
==========================================================

Una vulnerabilità in Hyper-V può consentire l'acquisizione di privilegi più elevati (2893986)
---------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 12 novembre 2013

**Versione:** 1.0

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità di Microsoft Windows che è stata segnalata privatamente. La vulnerabilità può consentire l'acquisizione di privilegi più elevati se un utente malintenzionato invia un parametro di funzione appositamente predisposto in un'hypercall da una macchina virtuale esistente in esecuzione sull'hypervisor. La vulnerabilità può anche consentire attacchi di tipo Denial of Service sull'host Hyper-V se l'utente malintenzionato invia un parametro di funzione appositamente predisposto in un'hypercall da una macchina virtuale esistente in esecuzione sull'hypervisor.

Questo aggiornamento per la protezione è considerato di livello importante per Windows 8 per sistemi x64 e Windows Server 2012. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento per la protezione risolve la vulnerabilità garantendo che Hyper-V disinfetti correttamente l'input dell'utente. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Per gli amministratori e le installazioni delle organizzazioni o gli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare l'aggiornamento quanto prima utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità degli aggiornamenti tramite il servizio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

Domande frequenti sull'aggiornamento
------------------------------------

<span></span>
**Il ruolo di Hyper-V non è attivato. È possibile installare questo aggiornamento?** **
**No. L'aggiornamento può essere installato solo sui sistemi operativi interessati con Hyper-V attivato. I sistemi con Hyper-V disattivato non sono interessati dalla vulnerabilità.

**Questo aggiornamento è disponibile anche se Hyper-V non è stato attivato?** **
**L'aggiornamento viene offerto tramite Aggiornamenti automatici solamente se l'aggiornamento automatico di Hyper-V è stato attivato. Ciò vale a prescindere dal fatto che il sistema operativo interessato sia stato installato o meno mediante l'opzione di installazione Server Core.

**Questo aggiornamento deve essere applicato alle macchine virtuali guest di Hyper-V?** **
**No. Questo aggiornamento non deve essere applicato alle macchine guest di Hyper-V ma solamente ai server che hanno Hyper-V attivato. L'aggiornamento viene offerto solo tramite Aggiornamenti automatici sul sistema Hyper-V Server; non è disponibile nelle macchine guest virtualizzate.

**Che cosa significa Hyper-V?**
Hyper-V è una tecnologia basata su hypervisor che rappresenta una funzionalità chiave di Windows Server 2008, Windows Server 2008 R2, Windows 8, Windows Server 2012, Windows 8.1 e Windows Server 2012 R2. Fornisce una piattaforma di virtualizzazione scalabile, affidabile e ad elevata disponibilità. Questo tipo di ambiente è utile perché è possibile creare e gestire delle macchine virtuali, consentendo così l'esecuzione di diversi sistemi operativi su un computer fisico isolandoli l'uno dall'altro.

**Che cos'è Hyper-V Server? È interessato anch'esso da questa vulnerabilità?** **
**Microsoft Hyper-V Server è un prodotto autonomo che offre una soluzione di virtualizzazione ottimizzata e affidabile che consente alle organizzazioni di migliorare l'utilizzo dei server e di ridurne i costi. Poiché Hyper-V Server è un prodotto dedicato autonomo, che contiene solamente Windows Hypervisor, il modello di driver Windows Server e i componenti per la virtualizzazione, occupa una quantità limitata di memoria e assicura un sovraccarico minimo.

Microsoft Hyper-V Server 2012 è interessato da questa vulnerabilità. Per essere protetti dalla vulnerabilità descritta in questo bollettino, è necessario applicare l'aggiornamento a Hyper-V Server. Ai clienti che hanno abilitato Aggiornamenti automatici verrà offerto automaticamente di installare l'aggiornamento.

Vulnerabilità legata al danneggiamento degli indirizzi - CVE-2013-3898
----------------------------------------------------------------------

<span></span>
Una vulnerabilità legata all'acquisizione di privilegi più elevati è stata riscontrata in Hyper-V su Windows 8 e Windows Server 2012. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe eseguire codice arbitrario come sistema in un'altra macchina virtuale (VM) dell'host Hyper-V condiviso. Un utente malintenzionato non potrebbe eseguire codice arbitrario sull'host Hyper-V, ma solo sulle macchine virtuali guest dello stesso host. La vulnerabilità potrebbe inoltre consentire attacchi di tipo Denial of Service in Hyper-V sulle stesse piattaforme, consentendo all'utente malintenzionato di bloccare la risposta o il riavvio dell'host Hyper-V.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2013-3898](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3898).

#### Fattori attenuanti

Microsoft non ha individuato alcun fattore attenuante per questa vulnerabilità.

#### Soluzioni alternative

Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.

#### Domande frequenti

**Qual è la portata o l'impatto di questa vulnerabilità?**
Si tratta di una vulnerabilità associata all'acquisizione di privilegi più elevati che potrebbe essere utilizzata per attacchi di tipo Denial of Service.

**Quali sono le cause di questa vulnerabilità?**
La vulnerabilità si presenta quando il valore di una struttura di dati non viene correttamente verificato, consentendo l'utilizzo di un indirizzo di memoria non valido.

**Un utente malintenzionato** **potrebbe assumere il controllo dell'host Hyper-V?**
No. L'host Hyper-V non è vulnerabile all'acquisizione di privilegi più elevati.

**Sono vulnerabili solo le macchine virtuali guest?**
No. Mentre solo le macchine virtuali guest sono vulnerabili all'acquisizione di privilegi più elevati, l'host Hyper-V è vulnerabile ad attacchi di tipo Denial of Service da una delle macchine virtuali guest in esecuzione. L'attacco di tipo Denial of Service ha inoltre un impatto su tutte le altre macchine virtuali sull'host, rendendole non disponibili finché il server non viene riavviato.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**
Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe causare l'arresto dell'host Hyper-V, con il conseguente arresto delle macchine virtuali guest. Un utente malintenzionato potrebbe inoltre eseguire potenzialmente codice su un'altra macchina virtuale guest. Perché questo accada, la macchina virtuale di destinazione deve trovarsi sullo stesso host della macchina virtuale da cui l'utente malintenzionato sta operando.

**In che modo un utente malintenzionato può sfruttare questa** **vulnerabilità?**
Un utente malintenzionato autenticato con privilegi di amministratore sulla macchina virtuale di origine potrebbe sfruttare questa vulnerabilità inviando un parametro di funzione appositamente predisposto in un'hypercall all'hypervisor dell'host.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**
I sistemi interessati con Hyper-V attivato sono principalmente esposti a questa vulnerabilità.

**Quali sono gli scopi dell'aggiornamento?**
L'aggiornamento risolve la vulnerabilità garantendo che Hyper-V disinfetti correttamente l'input dell'utente.

**Al momento del rilascio di questo bollettino, le informazioni sulla vulnerabilità erano disponibili pubblicamente?**
No. Microsoft ha ricevuto informazioni sulla vulnerabilità grazie a un'indagine coordinata.

**Al momento del rilascio di questo bollettino, erano già stati segnalati a Microsoft attacchi basati sullo sfruttamento di questa vulnerabilità?**
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco.
