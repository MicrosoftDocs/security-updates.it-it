Security Bulletin

Bollettino Microsoft sulla sicurezza MS06-077 - Importante
==========================================================

La vulnerabilità nel Servizio di installazione remota potrebbe consentire l'esecuzione di codice in modalità remota (926121)
----------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 12 dicembre 2006 | Aggiornamento: mercoledì 21 marzo 2007

**Versione:** 1.1

#### Riepilogo

**Destinatari:** Clienti che utilizzano Microsoft Windows

**Effetti della vulnerabilità:** Esecuzione di codice in modalità remota

**Livello di gravità:** Importante

**Raccomandazione:** È consigliabile installare l'aggiornamento per la protezione il prima possibile

**Aggiornamenti per la protezione sostituiti:** Nessuno

**Avvertenze:** L'[articolo della Microsoft Knowledge Base 926121](http://support.microsoft.com/kb/926121) descrive i problemi attualmente conosciuti che potrebbero verificarsi quando si installa questo aggiornamento per la protezione. L'articolo illustra inoltre le soluzioni consigliate in grado di risolvere questi problemi. Per ulteriori informazioni, vedere l'[articolo della Microsoft Knowledge Base 926121](http://support.microsoft.com/kb/926121).

Dettagli della vulnerabilità
----------------------------

<span></span>
#### Vulnerabilità del percorso scrivibile RIS - CVE-2006-5584:

Il Servizio di installazione remota attiva un servizio TFTP sul server che potrebbe consentire per impostazione predefinita a un utente anonimo di sovrascrivere potenzialmente i file del sistema operativo esistenti o caricare un file appositamente predisposto. In questo modo, un utente malintenzionato potrebbe compromettere le installazioni del sistema operativo offerte dal server RIS.

#### Fattori attenuanti relativi alla vulnerabilità del percorso scrivibile RIS - CVE-2006-5584:

-   L'utente malintenzionato deve essere in grado di effettuare l'invio tramite TFTP al server di installazione remoto per collocare un file o un programma appositamente predisposti.
-   Il Servizio di installazione remota non è installato per impostazione predefinita su Windows 2000 Service Pack 4.
-   Per i clienti che utilizzano il componente interessato dalla vulnerabilità, le procedure consigliate e le configurazioni predefinite standard dei firewall consentono di proteggere le reti dagli attacchi sferrati dall'esterno del perimetro aziendale. È consigliabile che i sistemi connessi a Internet abbiano un numero minimo di porte esposte.

#### Soluzioni alternative per la vulnerabilità del percorso scrivibile RIS - CVE-2006-5584:

Microsoft ha verificato le soluzioni alternative seguenti. Sebbene queste soluzioni alternative non eliminino la vulnerabilità sottostante, consentono di bloccare gli attacchi noti. Se una soluzione alternativa riduce la funzionalità, viene identificata nella sezione seguente.

-   **Configurare il servizio di TFTP come in sola lettura.**

    È possibile configurare il servizio di TFTP in sola lettura soltanto impostando un valore nel registro di sistema.

    **Nota** L'utilizzo non corretto dell'Editor del Registro di sistema potrebbe provocare problemi tali da rendere necessaria la reinstallazione del sistema operativo. Microsoft non garantisce la risoluzione di problemi dovuti a un utilizzo non corretto dell'Editor del Registro di sistema. È quindi necessario utilizzare questo strumento con grande attenzione. Per informazioni sulla modifica del Registro di sistema, vedere l'argomento relativo alla modifica delle chiavi e dei valori nella Guida dell'Editor del Registro di sistema (Regedit.exe) o gli argomenti relativi all'aggiunta ed eliminazione di informazioni nel Registro di sistema nonché alla modifica dei dati del Registro di sistema (regedit.exe).

    **Nota** Prima di apportare eventuali modifiche è consigliabile eseguire il backup del Registro di sistema.

    Per evitare che il server RIS accetti un accesso non autorizzato, aggiungere quanto segue al Registro di sistema utilizzando Regedit.exe.

    1.  Fare clic su **Start**, selezionare **Esegui**, digitare "regedit" (senza virgolette) e scegliere **OK**.
    2.  Nell'Editor del Registro di sistema creare la seguente chiave:
    3.  HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\TFTPD\\Parameters
    4.  Aggiungere il valore REG\_SZ: **Master**. Impostare il valore su **0**. Questo valore disattiva l'accesso TFTPD modificabile.
    5.  È necessario riavviare il servizio TFTP per rendere effettive le modifiche
    6.  Servizio TP:
    7.  Fare clic su **Start**, selezionare **Esegui**, digitare Esegui, digitare "**cmd**" (senza virgolette) e scegliere OK.
    8.  Nella finestra cmd digitare "Net stop TFTPD" (senza le virgolette) e premere **Invio**.

-   **Bloccare la porta 69 (UDP) a livello del firewall.**

    Questa porta viene utilizzata per stabilire una connessione con il componente interessato. Bloccando la porta UDP 69 a livello del firewall è quindi possibile evitare che i sistemi protetti dal firewall subiscano attacchi che tentino di sfruttare la vulnerabilità. Si consiglia di bloccare tutte le comunicazioni in ingresso non richieste provenienti da Internet per impedire gli attacchi che potrebbero utilizzare altre porte. Per ulteriori informazioni sulle porte, visitare il seguente [sito Web](http://go.microsoft.com/fwlink/?linkid=21312).

-   **Arrestare il servizio TFTP se non è necessario.**

    1.  Fare clic su **Start**, scegliere **Esegui**, digitare "cmd" (senza le virgolette) e scegliere **OK**.
    2.  Nella finestra cmd digitare "Net stop TFTPD" (senza le virgolette) e premere **Invio**.

    **Effetto della soluzione:** Se si disattiva il servizio TFTP, gli utenti non saranno in grado di caricare, visualizzare o scrivere il contenuto delle directory TFTP.

-   Per una maggiore protezione contro i tentativi di sfruttare questa vulnerabilità con attacchi basati sulla rete, attivare il filtro TCP/IP avanzato nei sistemi che lo supportano.

    Per bloccare tutto il traffico in ingresso non richiesto, è possibile attivare il filtro TCP/IP avanzato. Per ulteriori informazioni sulla procedura di configurazione del filtro TCP/IP, vedere l'[Articolo della Microsoft Knowledge Base 309798](http://support.microsoft.com/kb/309798).

-   Per una maggiore protezione contro i tentativi di sfruttare questa vulnerabilità con attacchi basati sulla rete, bloccare le porte interessate utilizzando IPSec nei sistemi interessati.

    Utilizzare IPSec (Internet Protocol Security) per proteggere le comunicazioni di rete. Per informazioni dettagliate su IPSec e sull'applicazione di filtri, vedere gli [articoli della Microsoft Knowledge Base 313190](http://support.microsoft.com/kb/313190) e [813878](http://support.microsoft.com/kb/813878).

#### Domande frequenti sulla vulnerabilità del percorso scrivibile RIS - CVE-2006-5584:

**Qual è la portata o l'impatto di questa vulnerabilità?**
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota. Se la vulnerabilità viene sfruttata, può consentire a un utente malintenzionato di assumere in remoto il controllo completo del sistema interessato, riuscendo quindi a installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi.

**Quali sono le cause di questa vulnerabilità?**
La vulnerabilità si presenta quando si consente l'accesso anonimo alla struttura di file di una build del sistema operativo ospitato tramite il servizio TFTP di RIS.

**Che cos'è RIS?**
Servizi di installazione remota (RIS) è una tecnologia di distribuzione basata su PXE (Pre-boot Execution Environment) che consente l'avvio dell'installazione di Windows in una rete.

**Quali capacità sono necessarie per sfruttare questa vulnerabilità?**
Qualsiasi utente anonimo in grado di inviare un eseguibile o un file appositamente predisposto al sistema RIS interessato può tentare di sfruttare questa vulnerabilità successivamente sui sistemi creati utilizzando la build del sistema operativo alterato.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**
Tutti i sistemi Windows 2000 Server Service Pack 4 con RIS installato sono principalmente a rischio.

**È possibile sfruttare questa vulnerabilità tramite Internet?**
Sì. Un utente malintenzionato può tentare di sfruttare questa vulnerabilità tramite Internet, ma devono innanzitutto disporre delle autorizzazioni di rete per ottenere l'accesso al RIS. Le configurazioni predefinite standard dei firewall e le procedure consigliate per la configurazione dei firewall consentono di proteggersi dagli attacchi provenienti da Internet.

**Quali sono gli scopi dell'aggiornamento?**
L'aggiornamento rimuove la vulnerabilità impedendo a utenti TFTP anonimi di scrivere sulla struttura di file della build del sistema operativo ospitato di RIS. L'aggiornamento aggiunge la chiave del Registro di sistema identificata nella sezione Soluzioni Alternative del bollettino.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private. Secondo le informazioni in possesso di Microsoft, al momento della pubblicazione del presente bollettino l'esistenza di questa vulnerabilità non era stata ancora divulgata pubblicamente.

**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco, né dell'esistenza di un codice di prova pubblicato.

    Per verificare i file installati dall'aggiornamento per la protezione, è anche possibile esaminare la seguente chiave del Registro di sistema:

    HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Updates\\Windows 2000\\SP5\\KB926121\\Filelist

    **Nota** Questa chiave di registro potrebbe non contenere un elenco completo dei file installati. Inoltre, quando un amministratore o un OEM integra o esegue lo slipstreaming dell'aggiornamento per la protezione nei file di origine dell'installazione di Windows, è possibile che questa chiave del Registro di sistema non venga creata correttamente.
