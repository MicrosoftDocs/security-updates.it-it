Security Bulletin

Bollettino Microsoft sulla sicurezza MS09-015 - Medio
=====================================================

Una vulnerabilità di varia natura in SearchPath può consentire l'acquisizione di privilegi più elevati (959426)
---------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 14 aprile 2009 | Aggiornamento: mercoledì 15 aprile 2009

**Versione:** 1.1

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità divulgata pubblicamente della funzione SearchPath di Windows che può consentire l'acquisizione di privilegi più elevati, se un utente scarica un file appositamente predisposto in una posizione specifica e apre quindi un'applicazione in grado di caricare il file in determinate circostanze.

Questo aggiornamento per la protezione è considerato di livello moderato per tutte le edizioni supportate di Windows XP, Windows Server 2003, Windows Vista e Windows Server 2008. In Microsoft Windows 2000 l'aggiornamento è classificato solo come una modifica al sistema di difesa. Ulteriori informazioni sono disponibili nella sezione **Domande frequenti sull'aggiornamento per la protezione**. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento per la protezione risolve la vulnerabilità modificando il modo in cui Windows carica i file dal desktop. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

Questo aggiornamento per la protezione risolve anche la vulnerabilità descritta nell'[advisory Microsoft sulla sicurezza 953818](http://technet.microsoft.com/security/advisory/953818).

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Agli amministratori e alle organizzazioni o agli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare l'aggiornamento utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità di questi tramite il servizio [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=it-it).

Domande frequenti sull'aggiornamento per la protezione
------------------------------------------------------

<span></span>
**Dove si trova la sezione Informazioni sui file?**  
La sezione Informazioni sui file è disponibile nell'[articolo della Microsoft Knowledge Base 959426](http://support.microsoft.com/kb/959426).

**Qual è la relazione tra il presente aggiornamento per la protezione e il bollettino MS09-014?**  
La vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati, [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540), descritta in questo bollettino sulla sicurezza interessa anche Internet Explorer. Microsoft consiglia agli utenti di installare anche l'aggiornamento per la protezione fornito dal bollettino [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014) Aggiornamento cumulativo per la protezione di Internet Explorer (963027).

**L'aggiornamento include modifiche di funzionalità relative alla sicurezza?**  
Sì. Oltre alle modifiche elencate nella sezione **Informazioni sulle vulnerabilità** del presente bollettino, l'aggiornamento contiene una modifica al sistema di difesa che introduce una nuova API, chiamata SetSearchPathMode.

**Che cosa consente di fare la modifica al sistema di difesa SetSearchPathMode?**  
Questa nuova API consente di attivare una modalità per singoli processi quando si utilizza la funzione SearchPath per trovare i file. In questo modo le applicazioni possono forzare l'esecuzione delle ricerche prima nella directory attuale, quindi in quelle di applicazione e sistema.

**Se si applica solo questo aggiornamento per la protezione, viene risolto il pericolo di varia natura descritto in CVE-2008-2540?**  
No. Occorre attivare separatamente anche il sistema di difesa specificato nell'Aggiornamento cumulativo per la protezione di Internet Explorer [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014). Si consiglia, tuttavia, di verificare che l'attivazione di questo sistema di difesa per il pericolo di varia natura sia adeguata al proprio ambiente.

Per attivare il sistema di difesa in Internet Explorer, attivare la chiave di controllo della funzionalità SearchPath, descritta nella sezione Domande frequenti sull'aggiornamento per la protezione in [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014). Per ulteriori informazioni, vedere la vulnerabilità identificata con il codice [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

**Perché Microsoft Windows 2000 è elencato nella tabella Software interessato senza che siano specificati l'impatto sulla protezione e il livello di gravità?**
Microsoft Windows 2000 è elencato senza che siano specificati l'impatto sulla protezione e il livello di gravità, poiché questo aggiornamento offre una soluzione diversa per Microsoft Windows 2000 rispetto ad altri sistemi operativi Windows supportati. Questo aggiornamento propone una modifica al sistema di difesa specifica per Microsoft Windows 2000.

**Perché la soluzione per il pericolo di varia natura in Microsoft Windows 2000 è diversa da quella di altre piattaforme? **
L'architettura per implementare la funzione SetSearchPathMode non esiste in Microsoft Windows 2000, pertanto la soluzione proposta per risolvere la vulnerabilità non è valida in questo sistema. Ciò comporterebbe la modifica di una parte significativa dell'architettura del sistema operativo Microsoft Windows 2000, non solo del componente interessato. Il risultato finale non sarebbe sufficientemente compatibile con Microsoft Windows 2000, dato che non vi sarebbe alcuna certezza che le applicazioni progettate per essere eseguite in Microsoft Windows 2000 possano continuare a funzionare sul sistema modificato. Tuttavia, Microsoft non è a conoscenza di alcun vettore di attacco valido in grado di sfruttare questa vulnerabilità in Microsoft Windows 2000.

A causa di queste modifiche all'architettura, in Microsoft Windows 2000 occorre applicare la modifica al sistema di difesa specifica per Microsoft Windows 2000, utilizzando la chiave del Registro di sistema HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode. Per istruzioni, vedere l'[articolo della Microsoft Knowledge Base 959426](http://support.microsoft.com/kb/959426). Per ulteriori informazioni, vedere l'articolo di MSDN, [Ordine di ricerca della libreria di collegamento dinamico](http://msdn.microsoft.com/library/ms682586.aspx).

**A cosa serve la funzione SearchPath?**  
La funzione SearchPath è utilizzata per cercare un file specifico in un determinato percorso. Per ulteriori informazioni, vedere l'articolo di MSDN [Funzione SearchPath](http://msdn.microsoft.com/library/aa365527.aspx).

**A cosa serve la funzione LoadLibrary?**  
La funzione LoadLibrary carica il modulo specificato nello spazio degli indirizzi del processo che richiama l'operazione. Il modulo specificato potrebbe causare il caricamento di altri moduli. Per ulteriori informazioni, vedere l'articolo di MSDN [Funzione LoadLibrary](http://msdn.microsoft.com/library/ms684175(vs.85).aspx).

Vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati - CVE-2008-2540  
----------------------------------------------------------------------------------------------
  
<span></span>  
Esiste una vulnerabilità di varia natura legata all'acquisizione di privilegi e relativa al modo in cui la funzione SearchPath di Windows individua e apre i file sul sistema. Un utente malintenzionato potrebbe sfruttare la vulnerabilità inducendo un utente a scaricare un file appositamente predisposto in una posizione specifica, quindi aprire un'applicazione che potrebbe caricare il file in determinate circostanze.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).
  
#### Fattori attenuanti della vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati - CVE-2008-2540
  
Microsoft non ha individuato alcun fattore attenuante per questa vulnerabilità.
  
#### Soluzioni alternative per la vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati - CVE-2008-2540
  
Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.
  
#### Domande frequenti sulla vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati - CVE-2008-2540
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Questa è una vulnerabilità di varia natura legata all'acquisizione di privilegi più elevati. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il pieno controllo in remoto del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi.
  
**Quali sono le cause di questa vulnerabilità?**    
Internet Explorer può aprire un file appositamente predisposto direttamente dal desktop, a seconda della posizione in cui vengono collocate le librerie di applicazioni all'interno del sistema Windows. Una vulnerabilità di varia natura che consenta ai file di essere scaricati sul sistema dell'utente senza richiedere conferma può far sì che Internet Explorer carichi accidentalmente un file scaricato dal desktop piuttosto che dal sistema Windows.
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Un utente malintenzionato in grado di sfruttare questa vulnerabilità potrebbe eseguire codice arbitrario, Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi.
  
**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Un utente malintenzionato potrebbe creare un file appositamente predisposto e indurre un utente a scaricarlo sul desktop. L'utente potrebbe avviare, quindi, un'applicazione che apre questo file appositamente predisposto. Tuttavia, un utente malintenzionato non avrebbe modo di obbligare gli utenti a scaricare tali file e posizionarli sul desktop.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
I sistemi più esposti sono quelli in cui gli utenti scaricano file e avviano applicazioni, ad esempio workstation e i terminal server.
  
**Quali sono gli scopi dell'aggiornamento?**    
L'aggiornamento modifica l'ordine con cui vengono cercate le librerie di applicazioni: la ricerca viene eseguita prima nella directory di sistema, quindi nelle directory di lavoro attuali. L'aggiornamento fornisce anche una modifica al sistema di difesa ovvero una nuova API, chiamata SetSearchPathMode, mediante la quale gli utenti più esperti possono stabilire l'ordine di ricerca desiderato per la funzione SearchPath. Per ulteriori informazioni su questa modifica al sistema di difesa, vedere anche la sezione **Domande frequenti sull'aggiornamento per la protezione**.
  
**Perché questa vulnerabilità e la vulnerabilità di varia natura legata all'esecuzione di codice in modalità remota presentano lo stesso codice CVE (CVE-2008-2540) nel bollettino MS09-014, Aggiornamento cumulativo per la protezione di Internet Explorer (963027)?**  
[CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) interessa sia Internet Explorer 7 per Windows XP Service Pack 2 e Windows XP Service Pack 3 e Internet Explorer 7 per Windows XP Professional x64 Edition e Windows XP Professional x64 Edition Service Pack 2 che la funzione SearchPath di Windows. Microsoft consiglia agli utenti di installare anche l'aggiornamento per la protezione fornito dal bollettino [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014) Aggiornamento cumulativo per la protezione di Internet Explorer (963027). Una modifica al sistema di difesa per questa vulnerabilità in questo aggiornamento, l'API denominata SetSearchPathMode, corrisponde anche alla soluzione illustrata nel bollettino [MS09-014](http://technet.microsoft.com/security/bulletin/ms09-014). Vedere anche la voce "L'aggiornamento include modifiche di funzionalità relative alla sicurezza?" delle Domande frequenti, nella sezione **Domande frequenti sull'aggiornamento per la protezione.**

**Verifica dell'applicazione dell'aggiornamento**

-   **Microsoft Baseline Security Analyzer**

    Per verificare se un aggiornamento per la protezione è stato applicato al sistema interessato, è possibile utilizzare lo strumento Microsoft Baseline Security Analyzer (MBSA). Per ulteriori informazioni, vedere la sezione **Strumenti e informazioni sul rilevamento e sulla distribuzione** nel presente bollettino.

-   **Verifica della versione dei file**

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. In alcuni casi, inoltre, è possibile che i file vengano rinominati durante l'installazione. Se le informazioni sul file o sulla versione non sono disponibili, utilizzare uno degli altri metodi disponibili per verificare l'installazione dell'aggiornamento.

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. -   **Verifica della versione dei file**

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. In alcuni casi, inoltre, è possibile che i file vengano rinominati durante l'installazione. Se le informazioni sul file o sulla versione non sono disponibili, utilizzare uno degli altri metodi disponibili per verificare l'installazione dell'aggiornamento.
