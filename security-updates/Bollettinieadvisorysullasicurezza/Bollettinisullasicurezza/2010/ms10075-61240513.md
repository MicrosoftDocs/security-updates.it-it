Security Bulletin

Bollettino Microsoft sulla sicurezza MS10-075 - Critico
=======================================================

Una vulnerabilità legata al servizio di condivisione di rete di Media Player può consentire l'esecuzione di codice in modalità remota (2281679)
-----------------------------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 12 ottobre 2010

**Versione:** 1.0

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità segnalata privatamente nel servizio di condivisione di rete di Microsoft Windows Media Player. Tale vulnerabilità può consentire l'esecuzione di codice in modalità remota nel momento in cui un utente malintenzionato invia un pacchetto RPC (Remote Procedure Call) appositamente predisposto a un sistema interessato. Tuttavia, l'accesso Internet ai file multimediali personali è disattivato per impostazione predefinita. In questa configurazione predefinita, la vulnerabilità può essere sfruttata solo da un utente malintenzionato all'interno della stessa subnet.

Questo aggiornamento per la protezione è considerato critico per le edizioni supportate di Windows 7 e importante per tutte le edizioni supportate di Windows Vista. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento per la protezione risolve la vulnerabilità correggendo il modo in cui il servizio di condivisione di rete di Windows Media Player gestisce pacchetti RTSP appositamente predisposti. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Per gli amministratori e le installazioni delle organizzazioni o gli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare immediatamente l'aggiornamento utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità degli aggiornamenti tramite il servizio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

Vulnerabilità legata a un errore di tipo use-after-free di RTSP - CVE-2010- il 3225  
-----------------------------------------------------------------------------------
  
<span></span>
Esiste una vulnerabilità nel servizio di condivisione di rete di Microsoft Windows Media Player che può consentire a un utente remoto di inviare un pacchetto di rete appositamente predisposto a un'istanza del servizio di streaming di rete e causare l'esecuzione di codice in modalità remota nel contesto dell'applicazione corrente.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2010-3225](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3225).
  
#### Fattori attenuanti della vulnerabilità legata a un errore di tipo use-after-free di RTSP - CVE-2010- il 3225
  
Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:
  
-   Per impostazione predefinita, l'accesso Internet ai file multimediali domestici è disattivato. In questa configurazione predefinita, la vulnerabilità può essere solo sfruttata da computer all'interno della stessa subnet.  
-   Sulle edizioni supportate di Windows Vista e Windows 7 Professional, Windows 7 Enterprise e Windows 7 Ultimate, il servizio di condivisione di rete di Windows Media Player non è attivato per impostazione predefinita.
  
#### Soluzioni alternative per la vulnerabilità legata a un errore di tipo use-after-free di RTSP - CVE-2010- il 3225
  
Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:
  
-   **Disattivare il servizio di condivisione di rete di Windows Media Player sui sistemiinteressati**
  
    **Metodo interattivo**
  
    **Nota** L'utilizzo non corretto dell'Editor del Registro di sistema potrebbe provocare problemi tali da rendere necessaria la reinstallazione del sistema operativo. Microsoft non garantisce la risoluzione di problemi dovuti a un utilizzo non corretto dell'Editor del Registro di sistema. È quindi necessario utilizzare questo strumento con grande attenzione. Per informazioni sulla modifica del Registro di sistema, vedere l'argomento relativo alla modifica delle chiavi e dei valori nella Guida dell'Editor del Registro di sistema (Regedit.exe) o gli argomenti relativi all'aggiunta ed eliminazione di informazioni nel Registro di sistema nonché alla modifica dei dati del Registro di sistema (Regedt32.exe).
  
    1.  Fare clic sul pulsante **Start**, scegliere **Esegui**, digitare **Regedit** nella casella **Apri** e fare clic su **OK**.  
    2.  Individuare e poi selezionare la seguente chiave del Registro di sistema:  
        HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\services\\WMPNetworkSvc  
    3.  Fare clic sul menu **File**, quindi selezionare **Esporta**.  
    4.  Nella finestra di dialogo **Esporta file del Registro di sistema**, digitare **WMPNetworkSvc\_Backup.reg** e selezionare **Salva**.  
        **Nota** Questa procedura crea per impostazione predefinita un backup della chiave del Registro di sistema nella cartella Documenti.  
    5.  Selezionare il valore **Avvia** sul riquadro a destra nell'Editor del Registro di sistema. Premere **Invio** per modificare il valore della chiave. Modificare il campo dei dati **Valore** a **4** e premere **Invio**.  
    6.  Da un prompt dei comandi di livello elevato, arrestare il servizio inserendo il comando seguente:  
        `sc stop WMPNetworkSvc`
  
    **Script di distribuzione gestita**
  
    1.  Eseguire una copia di backup del Registro di sistema utilizzando il comando seguente:  
        `regedit /e WMPNetworkSvc_Backup.reg HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WMPNetworkSvc`  
    2.  Salvare quanto segue in un file con estensione REG (ad esempio Disable\_WMPNetworkSvc.reg):  
        Editor del Registro di sistema di Windows versione 5.00  
        HKEY\_LOCAL\_MACHINE\\System\\CurrentControlSet\\Services\\service\_name\]  
        Start"=dword:00000004  
    3.  Eseguire lo script del Registro di sistema creato nel passaggio 2 nel computer di destinazione utilizzando il comando seguente:  
        `Regedit /s Disable_WMPNetworkSvc.reg`  
    4.  Da un prompt dei comandi di livello elevato, arrestare il servizio inserendo il comando seguente:  
        `sc stop WMPNetworkSvc`
  
    **Impatto della soluzione alternativa.** La condivisione di Windows Media Player sarà disattivata.
  
    **Per annullare il risultato della soluzione alternativa.**
  
    Utilizzo del metodo interattivo:
  
    1.  Fare clic sul pulsante **Start**, scegliere **Esegui**, digitare **Regedit** nella casella **Apri** e fare clic su **OK**.  
    2.  Fare clic sul menu **File**, quindi selezionare **Importa**.  
    3.  Nella finestra di dialogo **Importa file del Registro di sistema** , selezionare il file **WMPNetworkSvc\_Backup.reg** creato con il "Metodo Interattivo" precedente, quindi fare clic su **Apri**.  
    4.  Uscire dall'Editor del Registro di sistema e poi riavviare il computer.
  
    Utilizzo dello script di distribuzione gestita:
  
    1.  Eseguire lo script del Registro di sistema creato nel passaggio 2 nel computer di destinazione utilizzando il comando seguente:  
        `Regedit /s WMPNetworkSvc_Backup.reg`  
    2.  Riavviare il computer.
  
#### Domande frequenti sulla vulnerabilità legata a un errore di tipo use-after-free di RTSP - CVE-2010- il 3225
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota. Sfruttando questa vulnerabilità, un utente malintenzionato può eseguire codice nel contesto di protezione dell'account Network Service.
  
**Quali sono le cause di questa vulnerabilità?**    
La vulnerabilità è causata dal modo in cui il servizio di condivisione di rete di Microsoft Windows Media Player elabora i pacchetti RTSP (Real Time Streaming Protocol).
  
**In che modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Un utente malintenzionato può tentare di sfruttare la vulnerabilità creando e inviando un pacchetto RTSP appositamente predisposto a un sistema interessato.
  
**Che cos'è RTSP?**    
Il protocollo RTSP (Real Time Streaming Protocol) è un protocollo a livello di applicazione per il controllo della consegna dei dati con proprietà di tempo reale. Il protocollo RTSP fornisce una struttura estensibile per attivare la consegna controllata, su richiesta di dati in tempo reale, come audio e video. Le sorgenti di dati possono contenere sia feed di dati dal vivo e clip memorizzati. Questo protocollo è inteso controllare sessioni di consegna dei dati multiple, fornire un mezzo per la scelta dei canali di consegna, quali UDP, UDP multicast e TCP e fornire un mezzo per la scelta dei meccanismi consegna basati su RTP (RFC 1889). RTSP è definito in [RFC 2326](http://www.ietf.org/rfc/rfc2326.txt).
  
Il protocollo RTSP supporta le azioni di controllo del lettore, quali arresto, pausa, riavvolgimento e avvolgimento veloce di file multimediali di Windows. È possibile utilizzare il protocollo RTSP per inviare contenuti in streaming ai computer che eseguono Windows Media Player 9 Series o successivo o Windows Media Services 9 Series o successivo.
  
**Che cos'è un account NetworkService?**    
L'account Network Service è un account incorporato che offre un maggiore accesso a risorse e oggetti del gruppo Utenti. I servizi in esecuzione come account Network Service accedono alle risorse di rete utilizzando le credenziali dell'account del computer. Il nome effettivo dell'account è NT AUTHORITY\\NETWORK SERVICE. Per ulteriori informazioni, vedere l'articolo di MSDN NetworkService Account.
  
**Che cos'è il servizio di condivisione di rete di Windows Media Player?**    
Windows Media Player consente di condividere i file del catalogo multimediale con altri utenti sulla rete domestica ed esplorare i file presenti nel catalogo multimediale condivisi dagli altri utenti della rete. Windows Media Sharing utilizza il servizio di condivisione di Windows Media Player.
  
**Come si determina se il servizio di condivisione di rete di Windows Media Player è in esecuzione sul sistema?**    
Fare clic su **Start**, scegliere **Esegui**, digitare **Services.msc** e fare clic su **Continua**. Nel pannello di controllo Servizi, verificare se il servizio di condivisione di Windows Media Player ha lo stato "Avviato".
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Sfruttando questa vulnerabilità, un utente malintenzionato può assumere il controllo del sistema interessato nel contesto dell'account Network Service.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
I sistemi che eseguono il servizio di condivisione di rete di Windows Media Player (wmpnetwk.exe) per inviare in streaming i file multimediali sono principalmente a rischio.
  
**Quali sono gli scopi dell'aggiornamento?**    
L'aggiornamento per la protezione risolve la vulnerabilità correggendo il modo in cui il servizio di condivisione di rete di Windows Media Player gestisce pacchetti RTSP appositamente predisposti.

**Update Compatibility Evaluator e Application Compatibility Toolkit**
  
Gli aggiornamenti vanno spesso a sovrascrivere gli stessi file e le stesse impostazioni del Registro di sistema che sono necessari per eseguire le applicazioni. Ciò può scatenare delle incompatibilità e aumentare il tempo necessario per installare gli aggiornamenti per la protezione. Il programma [Update Compatibility Evaluator](http://technet.microsoft.com/it-it/library/cc766043(ws.10).aspx), incluso nell'[Application Compatibility Toolkit](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en), consente di semplificare il testing e la convalida degli aggiornamenti di Windows, verificandone la compatibilità con le applicazioni già installate.
  
L'Application Compatibility Toolkit (ACT) contiene gli strumenti e la documentazione necessari per valutare e attenuare i problemi di compatibilità tra le applicazioni prima di installare Microsoft Windows Vista, un aggiornamento di Windows, un aggiornamento Microsoft per la protezione o una nuova versione di Windows Internet Explorer nell'ambiente in uso.
