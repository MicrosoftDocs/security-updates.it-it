Security Bulletin

Bollettino Microsoft sulla sicurezza MS10-074 - Medio
=====================================================

La vulnerabilità legata alle classi di Microsoft Foundation può consentire l'esecuzione di codice in modalità remota (2387149)
------------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 12 ottobre 2010

**Versione:** 1.0

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità divulgata pubblicamente nella libreria Microsoft Foundation Class (MFC). La vulnerabilità può consentire l'esecuzione di codice in modalità remota se un utente è connesso con i diritti utente amministrativi e apre un'applicazione realizzata con la libreria MFC. Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere le stesse autorizzazioni dell'utente connesso. Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

Questo aggiornamento per la protezione è considerato di livello moderato per tutte le versioni supportate di Microsoft Windows. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento per la protezione risolve la vulnerabilità aggiungendo ulteriori verifiche alle richieste di modifica dei titoli di finestra. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Agli amministratori e alle organizzazioni o agli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare l'aggiornamento utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità di questi tramite il servizio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

Vulnerabilità legata all'overflow del buffer per le intestazioni dei documenti MFC di Windows - CVE-2010-3227  
-------------------------------------------------------------------------------------------------------------
  
<span></span>
Esiste una vulnerabilità legata all'esecuzione di codice in modalità remota nel modo in cui le intestazioni di finestra vengono gestite nelle applicazioni scritte mediante la libreria Microsoft Foundation Class (MFC). Mentre la vulnerabilità è localizzata in MFC ed è presente sui sistemi operativi interessati, può essere sfruttata solo se un utente malintenzionato remoto può influenzare l'intestazione di qualsiasi finestra in un'applicazione di MFC. Un utente malintenzionato in grado di sfruttare questa vulnerabilità può eseguire codice arbitrario nel contesto di protezione dell'utente corrente.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227).
  
#### Fattori attenuanti della vulnerabilità legata all'overflow del buffer per le intestazioni dei documenti MFC di Windows - CVE-2010-3227
  
Microsoft non ha individuato alcun fattore attenuante per questa vulnerabilità.
  
#### Soluzioni alternative della vulnerabilità legata all'overflow del buffer per le intestazioni dei documenti MFC di Windows - CVE-2010-3227
  
Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.
  
#### Domande frequenti sulla vulnerabilità legata all'overflow del buffer per le intestazioni dei documenti MFC di Windows - CVE-2010-3227
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Questa vulnerabilità è legata all'esecuzione di codice in modalità remota nel contesto di protezione dell'utente attualmente connesso.
  
**Quali sono le cause di questa vulnerabilità?**    
La vulnerabilità è causata dal modo in cui MFC gestisce le richieste dell'applicazione di rinominare le intestazioni di finestra.
  
**Che cos'è la libreria di Microsoft Foundation Class (MFC)?**    
La libreria di Microsoft Foundation Class (MFC) è una struttura di applicazione di programmazione in Microsoft Windows. Scritto in C ++, MFC fornisce buona parte del codice necessario a eseguire le operazioni standard, quali la gestione delle finestre, dei menu e delle finestre di dialogo ed eseguire input/output di base o memorizzare o raccolte di oggetti di dati.
  
**Questo aggiornamento sarà offerto anche se non sono presenti vettori di attacco idonei sul sistema?**    
Sì, questo aggiornamento sarà offerto a tutti i sistemi che eseguono il software secondo quanto delineate nella sezione Software interessato. Se nessuna applicazione MFC è attualmente installata su un sistema, Microsoft consiglia agli utenti di installare questo aggiornamento come misura di difesa approfondita, nel caso in cui un vettore di attacco venga introdotto o reso noto in un momento successivo.
  
**Vi sono applicazioni Microsoft che possano essere utilizzate come vettore di attacco?**    
Microsoft ha svolto un'accurata ricerca su tutte le applicazioni, ma non ha riscontrato alcuna applicazione Microsoft che possa essere utilizzata con successo come vettore di attacco.
  
**Microsoft è a conoscenza di eventuali applicazioni di terzi che possano essere utilizzate come vettore di attacco?**    
Sì. Microsoft è a conoscenza di un'applicazione di terzi che può essere utilizzata come vettore di attacco. Lo sfruttamento di questa vulnerabilità mediante questo vettore di attacco di terzi richiederebbe che un utente apra un file appositamente predisposto con l'applicazione MFC, quindi faccia doppio clic su una cartella all'interno dell'applicazione MFC.
  
**Ho scritto un'applicazione utilizzando le classi di Microsoft Foundation. La mia applicazione può essere utilizzata come vettore di attacco?**    
L'applicazione può essere un vettore di attacco se tutte le condizioni seguenti sono vere:
  
-   L'applicazione utilizza la libreria Microsoft Foundation Class (MFC)  
-   L'applicazione imposta le intestazioni di finestra in base alle informazioni che possono essere controllate da un utente remoto
  
**Nota** Microsoft consiglia che le applicazioni MFC si colleghino dinamicamente alle versioni di MFC fornite con Windows. Se l'applicazione è dotata di una versione MFC vulnerabile e si collega staticamente a essa, si consiglia caldamente di aggiornare l'applicazione in modo che si colleghi dinamicamente a una versione MFC fornita con Windows.
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere le stesse autorizzazioni dell'utente connesso. Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.
  
**In che modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Per sfruttare questa vulnerabilità, il sistema di destinazione dovrebbe avere installata un'applicazione MFC che possa essere utilizzata come vettore di attacco. Le applicazioni MFC che possono essere utilizzate come vettore di attacco devono fornire un modo con il quale un utente remoto possa influenzare le intestazioni di finestra all'interno dell'applicazione MFC. Esempi possono essere la modifica di un'intestazione di finestra in base al contenuto di un file aperto con l'applicazione MFC oppure una modifica di intestazione di finestra basata sull'input di rete, come nel caso di un'applicazione di chat o di browser web. In funzione dello specifico vettore di attacco, un utente malintenzionato può dover persuadere un utente ad aprire un file appositamente predisposto con un'applicazione MFC specifica, installata sul sistema dell'utente ed eseguire azioni specifiche. In un vettore di attacco diverso, come un'applicazione di chat, può essere sufficiente per l'utente malintenzionato modificare il nome di una chat-room o utilizzare un handle appositamente predisposto mentre contatta l'utente.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
Le workstation e i terminal server sono principalmente a rischio se sono installare applicazioni MFC che consentono a un utente remoto di influenzare le intestazioni di finestra all'interno dell'applicazione MFC. I server possono essere maggiormente a rischio se gli amministratori consentono agli utenti di accedere ai server ed eseguire programmi MFC. Tuttavia, le procedure consigliate scoraggiano fortemente questa attribuzione di privilegi.
  
**Quali sono gli scopi dell'aggiornamento?**    
L'aggiornamento risolve la vulnerabilità aggiungendo verifiche aggiuntive alle richieste di modifica delle intestazioni di finestra.
  
**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**    
Sì. Le informazioni sulla vulnerabilità sono state divulgate pubblicamente. Questa vulnerabilità è stata identificata con il codice [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227).
  
**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**    
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco.

**Verifica dell'applicazione dell'aggiornamento**

Per ulteriori informazioni, vedere la sezione **Strumenti e informazioni sul rilevamento e sulla distribuzione** nel presente bollettino.

-   **Verifica della versione dei file**

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**.
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
