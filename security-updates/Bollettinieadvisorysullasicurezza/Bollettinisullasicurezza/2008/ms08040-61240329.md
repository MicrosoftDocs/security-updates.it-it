Security Bulletin

Bollettino Microsoft sulla sicurezza MS08-040 - Importante
==========================================================

Alcune vulnerabilità in Microsoft SQL Server possono consentire l'acquisizione di privilegi più elevati (941203)
----------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 8 luglio 2008 | Aggiornamento: mercoledì 18 marzo 2009

**Versione:** 1.9

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve quattro vulnerabilità segnalate privatamente a Microsoft. La più pericolosa di tali vulnerabilità può consentire a un utente malintenzionato di eseguire del codice e di assumere il controllo completo di un sistema interessato. L'utente malintenzionato autenticato potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

Questo aggiornamento per la protezione è considerato di livello importante per le versioni supportate di SQL Server 7.0, SQL Server 2000, SQL Server 2005, Microsoft Data Engine (MSDE) 1.0, Microsoft SQL Server 2000 Desktop Engine (MSDE 2000), Microsoft SQL Server 2005 Express Edition, Microsoft SQL Server 2000 Desktop Engine (WMSDE) e Windows Internal Database (WYukon). Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento per la protezione risolve le vulnerabilità modificando il modo in cui SQL Server gestisce il riutilizzo delle pagine, allocando più memoria per la funzione di conversione, convalidando i file su disco prima di caricarli e convalidando le istruzioni INSERT. Per ulteriori informazioni sulle vulnerabilità, cercare la voce specifica nella sottosezione Domande frequenti della sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Microsoft consiglia di installare l'aggiornamento quanto prima.

**Problemi noti.** L'[articolo della Microsoft Knowledge Base 941203](http://support.microsoft.com/kb/941203) descrive i problemi attualmente conosciuti che potrebbero verificarsi installando il presente aggiornamento per la protezione. L'articolo illustra inoltre le soluzioni consigliate in grado di risolvere questi problemi. Per ulteriori informazioni sulla installazione degli aggiornamenti, vedere la sottosezione **Domande frequenti (FAQ) relative a questo aggiornamento per la protezione** in questa sezione.

#### Software interessato e Software non interessato

Il seguente software è stato sottoposto a test per determinare quali versioni o edizioni siano interessate. Le altre versioni o edizioni non sono interessate dalla vulnerabilità o sono al termine del ciclo di vita del supporto. Per informazioni sulla disponibilità del supporto per la versione o l'edizione del software in uso, visitare il [sito Web Ciclo di vita del supporto Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

A seconda della versione o dell'edizione del software in uso, l'utente dovrà scegliere tra un aggiornamento del software GDR o QFE e tra i relativi collegamenti sottostanti, se desidera installare manualmente il proprio aggiornamento dall'Area download Microsoft. Per ulteriori informazioni su come determinare quale aggiornamento è necessario installare nel proprio sistema, vedere la sottosezione **Domande frequenti sull'aggiornamento per la protezione** di questa sezione.

Domande frequenti sull'aggiornamento per la protezione  
------------------------------------------------------
  
<span></span>
**Perché Microsoft ha rivisto questo bollettino in data 12 agosto 2008?**    
Microsoft ha rivisto questo bollettino per comunicare una modifica nel codice di installazione dell'aggiornamento per la protezione di SQL Server 2005 Service Pack 2 finalizzata alla risoluzione di un problema di installazione. La modifica riguarda esclusivamente il codice di installazione. Non sono previsti aggiornamenti per la protezione di file binari. Sebbene gli amministratori WSUS (Windows Server Update Services) considereranno questa modifica come una nuova revisione all'aggiornamento, i clienti che hanno già installato l'aggiornamento per la protezione non dovranno reinstallarlo.
  
**Perché Microsoft ha rivisto questo bollettino in data 23 luglio 2008?**    
Questo bollettino è stato rivisto per aggiungere le note sulle informazioni per la rimozione per Microsoft SQL Server 2000 Desktop Engine (WMSDE) e Windows Internal Database (WYukon) chiarendo che la rimozione dell'aggiornamento per la protezione per WMSDE o WYukon causerà anche la completa rimozione dal sistema dell'istanza di WMSDE o WYukon.
  
**Perché Microsoft ha rivisto questo bollettino in data 18 luglio 2008?**    
Questo bollettino è stato rivisto per comunicare che è stata apportata una modifica relativa al rilevamento nel modo in cui Windows Server Update Services (WSUS) offre l'aggiornamento per Microsoft SQL Server 2000 Desktop Engine (WMSDE). Si tratta di una modifica solo al rilevamento. Non sono previste modifiche ai file binari.
  
**Perché Microsoft ha rivisto questo bollettino in data 11 luglio 2008?**    
Questo bollettino è stato rivisto per comunicare che la sezione **Problemi attualmente conosciuti illustrati nel presente aggiornamento per la protezione**, all'interno dell'[articolo della Microsoft Knowledge Base 948110](http://support.microsoft.com/kb/948110) associato, è stata aggiornata per descrivere un problema di rivelamento degli aggiornamenti SKU per SQL Server 2000 Desktop Engine (WMSDE).
  
**Per la mia versione di SQL è disponibile sia un aggiornamento GDR che un aggiornamento QFE. Come posso sapere quale aggiornamento è adatto per il mio sistema?**    
In primo luogo è necessario accertare il numero della versione di SQL Server in uso. Per ulteriori informazioni su come determinare il numero della versione di SQL Server in uso, vedere l'[articolo della Microsoft Knowledge Base 321185](http://support.microsoft.com/kb/321185).
  
In secondo luogo è necessario individuare nella tabella sottostante l'intervallo che prevede il numero della versione di SQL Server in uso. L'aggiornamento associato a tale intervallo è quello che si deve installare.
  
**Nota** Se il numero della versione di SQL Server in uso non rientra in alcuno degli intervalli previsti dalla tabella sottostante, significa che la versione di SQL Server in uso non è più supportata. Per poter applicare questo aggiornamento per la protezione e quelli futuri, è necessario aggiornare il sistema in uso con il service pack o il prodotto SQL più recenti.

**Perché l'aggiornamento software QFE KB948108 non è disponibile per il sistema SQL Server 2005 Service Pack 2?**  
Come illustrato in modo dettagliato [nell'articolo della Microsoft Knowledge Base 935897](http://support.microsoft.com/kb/935897), le correzioni rapide per SQL Server 2005 sono cumulative. In genere, le correzioni rapide hanno una finestra di verifica minore rispetto alle GDR (General Distribution Release). I clienti che hanno installato il [pacchetto di aggiornamento cumulativo 7 per SQL Server 2005 Service Pack 2](http://support.microsoft.com/kb/949095) di aprile o il [pacchetto di aggiornamento cumulativo 8 per SQL Server 2005 Service Pack 2](http://support.microsoft.com/kb/951217) di giugno non potranno ususfruire dell'aggiornamento software QFE KB948108, perché gli aggiornamenti per la protezione disponibili tramite l'aggiornamento software QFE KB948108 erano già contenuti negli aggiornamenti cumulativi di aprile e giugno. Questo aggiornamento è comunque valido per tutti i clienti che non hanno applicato l'aggiornamento cumulativo di aprile o giugno per SQL Server 2005 Service Pack 2.

**Le tabelle di riferimento per SQL Server 2000 e MSDE 2000** **nella sezione Informazioni sull'aggiornamento per la protezione mostrano quali aggiornamenti richiedono un riavvio. Come posso sapere se il mio aggiornamento richiede un riavvio?**  
SQL Server 2000 e MSDE 2000 dispongono talvolta di servizi dipendenti. I servizi dipendenti non vengono riavviati quando il servizio da cui dipendono viene arrestato e riavviato. Per questo motivo è necessario eseguire un riavvio, dopo avere installato un aggiornamento, per garantire che i servizi dipendenti vengano anch'essi avviati. Per informazioni su come determinare se il sistema in uso prevede dei servizi dipendenti, vedere l'[articolo della Microsoft Knowledge Base 953741](http://support.microsoft.com/kb/953741).

**Gli aggiornamenti verranno applicati alle istanze disabilitate di SQL Server 2000, MSDE 2000 e SQL Server 2005?**  
No. Verranno aggiornate dal programma di installazione soltanto quelle istanze di SQL Server 2000, MSDE 2000 e SQL Server 2005 che non hanno un avvio di tipo **disabilitato**. Per ulteriori informazioni sui servizi disabilitati, vedere l'[articolo della Microsoft Knowledge Base 953740](http://support.microsoft.com/kb/953740).

**L'aggiornamento per SQL Server 7.0 Service Pack 4 e Microsoft Data Engine (MSDE) 1.0 Service Pack 4 richiede l'installazione manuale?**  
Sì. È necessario installare manualmente l'aggiornamento, seguendo le istruzioni contenute nel file Readme associato al file eseguibile autoestraente fornito con l'aggiornamento. Per ulteriori informazioni sull'installazione manuale di SQL Server 7.0 Service Pack 4 e Microsoft Data Engine (MSDE) 1.0 Service Pack 4, vedere l'[articolo della Microsoft Knowledge Base 953743](http://support.microsoft.com/kb/953743). Per le istruzioni di installazione aggiuntive, vedere la sottosezione Informazioni sull'aggiornamento per la protezione per SQL Server 7.0 e Microsoft Data Engine (MSDE) nella sezione di **Informazioni sull'aggiornamento**.

**Se si utilizza una versione precedente del software discusso nel presente bollettino, che cosa è necessario fare?**  
I prodotti software elencati in questo bollettino sono stati sottoposti a test per determinare quali versioni o edizioni sono interessate dalla vulnerabilità. Le altre versioni o edizioni sono al termine del ciclo di vita del supporto. Per informazioni sulla disponibilità del supporto per la versione o l'edizione del software in uso, visitare il [sito Web Ciclo di vita del supporto Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Per evitare di esporre i propri sistemi a potenziali vulnerabilità, si raccomanda ai clienti che utilizzano versioni o edizioni precedenti del software di eseguire quanto prima la migrazione a versioni o edizioni supportate. Per ulteriori informazioni sul ciclo di vita dei prodotti Windows, visitare il sito Web [Ciclo di vita del supporto Microsoft](http://go.microsoft.com/fwlink/?linkid=21742). Per ulteriori informazioni sul periodo di supporto esteso per gli aggiornamenti per la protezione di queste versioni o edizioni del software, visitare il sito Web del [Servizio Supporto Tecnico Clienti Microsoft](http://go.microsoft.com/fwlink/?linkid=33328).

Vulnerabilità legata al riutilizzo delle pagine di memoria – CVE-2008-0085
--------------------------------------------------------------------------

<span></span>
È stata riscontrata una vulnerabilità che consente l'intercettazione di informazioni personali nella procedura con cui SQL Server gestisce il riutilizzo delle pagine di memoria. Un utente malintenzionato dotato di diritti di accesso come operatore di database potrebbe sfruttare questa vulnerabilità per accedere ai dati dei clienti.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2008-0085](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0085).

#### Fattori attenuanti dell'intercettazione di informazioni personali dovuta alla vulnerabilità legata al riutilizzo delle pagine di memoria - CVE-2008-0085

Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:

-   Per sfruttare questa vulnerabilità l'utente malintenzionato deve avere accesso come operatore di database al database, al registro, ai file di backup del database o ai file di backup del registro.

#### Soluzioni alternative per l'intercettazione di informazioni personali dovuta alla vulnerabilità legata al riutilizzo delle pagine di memoria - CVE-2008-0085

Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:

-   **Attiva conformità criteri comuni su SQL 2005 Service Pack 2**

    Per attivare la [conformità ai criteri comuni](http://msdn.microsoft.com/it-it/library/bb326650.aspx), eseguire il seguente script connettendosi a SQL Server in qualità di amministratore. Dopo avere eseguito lo script, riavviare SQL Server.

    `sp_configure 'show advanced options', 1;GORECONFIGURE;GOsp_configure 'common criteria compliance enabled', 1;GORECONFIGUREGO`

    **Effetto della soluzione alternativa.** L'attivazione dell'opzione di conformità ai criteri comuni produce una leggera riduzione delle prestazioni.

    **Per annullare il risultato della soluzione alternativa.** Per disattivare la conformità ai criteri comuni, eseguire il seguente script connettendosi a SQL Server in qualità di amministratore. Dopo avere eseguito lo script, riavviare SQL Server.

    `sp_configure 'show advanced options', 1;GORECONFIGURE;GOsp_configure 'common criteria compliance enabled', 0;GORECONFIGUREGO`

#### Domande frequenti sull'intercettazione di informazioni personali dovuta alla vulnerabilità legata al riutilizzo delle pagine di memoria - CVE-2008-0085

**Qual è la portata o l'impatto di questa vulnerabilità?**  
È stata riscontrata una vulnerabilità che consente l'intercettazione di informazioni personali nella procedura con cui SQL Server gestisce il riutilizzo delle pagine di memoria. Un utente malintenzionato dotato di diritti di accesso come operatore di database potrebbe sfruttare questa vulnerabilità per accedere a dati di clienti, la cui visualizzazione gli sarebbe altrimenti preclusa.

**Quali sono le cause di questa vulnerabilità?**  
La vulnerabilità è originata dal modo in cui SQL Server gestisce il riutilizzo delle pagine di memoria. Quando deve riallocare della memoria, SQL Server non riesce a inizializzare le pagine di memoria.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**  
Un utente malintenzionato potrebbe leggere il contenuto delle pagine di memoria che includono i dati della sessione di un altro utente.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**  
Un utente malintenzionato dotato di diritti di accesso come operatore di database potrebbe comporre le pagine di memoria non inizializzate, che includono i dati della sessione di un altro utente, collocando un backup in una posizione controllata dall'utente malintenzionato.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**  
I sistemi più esposti sono quelli in cui SQL Server viene utilizzato da entità non attendibili.

**Quali sono gli scopi dell'aggiornamento? **
L'aggiornamento elimina la vulnerabilità modificando il modo in cui SQL Server gestisce il riutilizzo delle pagine di memoria.

**Come posso eliminare i dati residui dai miei database? **
La nuova stored procedure di sistema illustrata di seguito ha lo scopo di eliminare i dati residui dallo spazio libero delle pagine di database negli ambienti in cui è a rischio la sicurezza fisica dei dati o dei file di backup:

`sp_clean_db_free_space`

Questa stored procedure rimuove le informazioni residue lasciate sulle pagine di database dalle procedure di modifica dei dati di SQL Server. La procedura pulisce tutte le pagine di tutti i file del database.

`sp_clean_db_free_space@dbname = 'database_name',[ @cleaning_delay = 'delay_in_seconds' ]`

**Argomenti**

`@dbname =  'database_name',`

-   Il nome della database da pulire. *dbname* è **nvarchar(384)** e non può essere NULLO.

`[ @cleaning_delay = 'delay_in_seconds' ]`

-   Per ridurre l'impatto sul sistema I/O, è possibile immettere un ritardo tra la pulizia delle pagine. *cleaning\_delay* è **int** con un valore predefinito di 0.

L'esempio seguente pulisce tutte le informazioni residue dal database di AdventureWorks.

`USE MasterGOEXEC sp_clean_db_free_space@dbname = 'AdventureWorks';`

Sovraccarico del buffer di conversione - CVE-2008-0086
------------------------------------------------------

<span></span>
Nella procedura di conversione di SQL Server esiste una vulnerabilità che può consentire a un utente malintenzionato autenticato di acquisire privilegi più elevati. Sfruttando tale vulnerabilità, un utente malintenzionato può eseguire del codice e assumere il controllo completo del sistema.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2008-0086](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0086).

#### Fattori attenuanti del sovraccarico del buffer di conversione - CVE-2008-0086

Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:

-   Un utente malintenzionato essere in possesso di credenziali di accesso Windows valide con autorizzazioni utente a SQL per tentare di sfruttare questa vulnerabilità.

#### Soluzioni alternative per il sovraccarico del buffer di conversione - CVE-2008-0086

Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:

-   Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.

#### Domande frequenti sul sovraccarico del buffer di conversione - CVE-2008-0086

**Qual è la portata o l'impatto di questa vulnerabilità?**  
È stata riscontrata una vulnerabilità legata all'acquisizione di privilegi più elevati nella procedura con cui SQL Server converte le espressioni SQL da un tipo di dati a un altro. Questa vulnerabilità può consentire a un utente malintenzionato di eseguire del codice e assumere il controllo completo del sistema. L'utente malintenzionato autenticato potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

**Quali sono le cause di questa vulnerabilità?**  
La funzione di conversione di SQL Server non controlla in modo sufficiente le stringhe di input. Ne deriva un sovraccarico del buffer, che consente a un utente malintenzionato autenticato di eseguire del codice non autorizzato.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**  
Sfruttando questa vulnerabilità, un utente malintenzionato può acquisire privilegi più elevati in un sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**  
Un utente malintenzionaton autenticato può creare una query che definisce la funzione di conversione con un'espressione appositamente predisposta, provocando l'overflow della funzione e consentendo l'esecuzione di codice.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**  
I sistemi più esposti sono quelli che utilizzano SQL Server 2000 Service Pack 4 e Microsoft SQL Server 2000 Desktop Engine (WMSDE).

**Quali sono gli scopi dell'aggiornamento? **
L'aggiornamento rimuove la vulnerabilità allocando più memoria alla funzione di conversione.

-   Un utente malintenzionato essere in possesso di credenziali di accesso Windows valide con autorizzazioni utente a SQL per tentare di sfruttare questa vulnerabilità.
-   Per impostazione predefinita, SQL Server 2005 non viene eseguito con privilegi di sistema.
-   Per impostazione predefinita, SQL Server 2005 richiede all'amministratore di scegliere un account utente di dominio durante l'installazione. Se SQL Server 2005 è stato configurato dal programma di installazione per essere eseguito come un account di dominio privo di privilegi, è meno probabile che l'account di dominio risulti influenzato da tentativi di sfruttare questa vulnerabilità.

#### Soluzioni alternative per la vulnerabilità legata al sovraccarico del buffer di SQL Server – CVE-2008-0106

Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. #### Domande frequenti relative alla vulnerabilità legata al sovraccarico del buffer di SQL Server – CVE-2008-0106

**Qual è la portata o l'impatto di questa vulnerabilità?**  
È stata riscontrata una vulnerabilità legata all'acquisizione di privilegi più elevati nella procedura con cui SQL Server gestisce la memoria durante l'elaborazione delle istruzioni INSERT. Questa vulnerabilità può consentire a un utente malintenzionato autenticato di eseguire del codice e assumere il controllo completo del sistema. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

**Quali sono le cause di questa vulnerabilità?**  
SQL Server non controlla in modo sufficiente le istruzioni INSERT prima di elaborarle. Ne deriva un sovraccarico del buffer, che consente a un utente malintenzionato autenticato di eseguire del codice non autorizzato.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**  
Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe acquisire privilegi più elevati nel sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**  
Un utente malintenzionato autenticato può creare delle istruzioni INSERT che provocano un sovraccarico del buffer e corrompono la memoria, consentendo in tal modo l'esecuzione di codice.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**  
I sistemi più esposti sono quelli che utilizzano SQL Server 2005.

**Quali sono gli scopi dell'aggiornamento? **
L'aggiornamento elimina la vulnerabilità convalidando le istruzioni INSERT.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente? **
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private. Secondo le informazioni in possesso di Microsoft, al momento della pubblicazione del presente bollettino l'esistenza di questa vulnerabilità non era stata ancora divulgata pubblicamente. Il presente bollettino sulla sicurezza risolve questa vulnerabilità nonché altri problemi scoperti tramite ricerche interne.

**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità? **
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco, né dell'esistenza di un codice di prova pubblicato.
