Security Bulletin

Bollettino Microsoft sulla sicurezza MS05-044 - Medio
=====================================================

Una vulnerabilità in Windows FTP può consentire un'alterazione del percorso per il trasferimento di file (905495)
-----------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 11 ottobre 2005 | Aggiornamento: mercoledì 26 ottobre 2005

**Versione:** 1.0

**Data di pubblicazione:** 11 ottobre 2005
**Versione:** 1.0

**Destinatari:** clienti che utilizzano Microsoft Windows.

**Effetti della vulnerabilità:** alterazione.

**Livello di gravità:** moderato.

**Raccomandazioni:** è consigliabile installare l'aggiornamento per la protezione.

Dettagli della vulnerabilità
----------------------------

<span></span>
#### Vulnerabilità nel client FTP - CAN-2005-2126:

Nel client Windows FTP esiste una vulnerabilità legata all'alterazione che può consentire a un utente malintenzionato di modificare il percorso di destinazione specificato per il trasferimento di file, quando un client ha scelto di trasferire un file mediante FTP. In questo modo, l'utente malintenzionato potrebbe registrare il file in qualsiasi file system sul sistema interessato.

#### Fattori attenuanti della vulnerabilità nel client FTP- CAN-2005-2126:

-   Anche se un utente malintenzionato riuscisse a convincere gli utenti a visitare un server FTP che ospita file con nomi file appositamente predisposti, il trasferimento di file non avverrebbe comunque. È necessaria l'interazione dell'utente prima che i file possano essere trasferiti al sistema interessato.
-   Se durante il trasferimento mediante FTP l'utente malintenzionato tenta di sostituire un file esistente sul sistema interessato, si riceve un messaggio di avviso di tipo "Sovrascrivere il file?". Il file viene salvato solo se l'utente ne consente il salvataggio dopo aver ricevuto il messaggio di avviso.
-   Per impostazione predefinita, l'impostazione di Internet Explorer "Attiva la visualizzazione della cartella per i siti FTP" è disabilitata su tutte le versioni del sistema operativo interessato. L'utente malintenzionato può riuscire nel suo intento solo se è stata abilitata manualmente l'impostazione di Internet Explorer "Attiva la visualizzazione della cartella per i siti FTP" sul sistema interessato.

#### Soluzioni alternative per la vulnerabilità nel client FTP - CAN-2005-2126:

Microsoft ha verificato le soluzioni alternative seguenti. Sebbene queste soluzioni alternative non eliminino la vulnerabilità sottostante, consentono di bloccare gli attacchi noti. Se una soluzione alternativa riduce la funzionalità, viene identificata nella sezione seguente.

-   **Non scaricare file da server FTP non attendibili**

    È possibile ridurre il rischio di attacco scaricando i file solo da server FTP attendibili.

#### Domande frequenti relative alla vulnerabilità nel client FTP - CAN-2005-2126:

**Qual è la portata o l'impatto di questa vulnerabilità?**
Si tratta di una vulnerabilità legata all'alterazione. Tale vulnerabilità può consentire a un utente malintenzionato di modificare il percorso di destinazione specificato per il trasferimento di file, quando un client ha scelto di trasferire un file mediante FTP.

**Quali sono le cause di questa vulnerabilità?**
Il client Windows FTP non convalida in modo corretto i nomi file ricevuti dai server FTP.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**
Un utente malintenzionato potrebbe sfruttare questa vulnerabilità salvando i file in percorsi specifici su un sistema interessato per consentire altri attacchi. L'utente malintenzionato potrebbe, ad esempio, salvare un file eseguibile nella cartella "Avvio". In questo modo, sarebbe possibile eseguire il file trasferito al successivo accesso dell'utente.

**Quali capacità sono necessarie per sfruttare questa vulnerabilità?**
Qualsiasi utente in grado di indurre qualcuno a visitare e trasferire file da un server FTP che ospita file con nomi file appositamente predisposti.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**
Un utente malintenzionato può sfruttare la vulnerabilità memorizzando un file su un server FTP con un nome appositamente predisposto. Il nome file deve essere creato in modo tale da eludere la convalida del nome file fornita dal client FTP e associarlo a un percorso valido sul computer degli utenti. L'utente malintenzionato potrebbe poi indurre l'utente a scaricare tale file.

**È possibile sfruttare questa vulnerabilità automaticamente visitando un server FTP?**
No. L'interazione dell'utente è necessaria per trasferire e salvare il file mediante FTP.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**
La vulnerabilità richiede che l'utente sia collegato a un server FTP e trasferisca file dallo stesso. Quindi, qualsiasi sistema nel quale si verificano spesso trasferimenti FTP, ad esempio le workstation, è maggiormente esposto ai rischi derivanti da questa vulnerabilità. Nei sistemi che non vengono comunemente utilizzati per visitare server FTP, come la maggior parte dei server, il livello di rischio è ridotto.

**Quali sono gli scopi dell'aggiornamento?**
L'aggiornamento elimina la vulnerabilità modificando il modo il cui il client Windows FTP convalida i nomi file ricevuti dai server FTP.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**
Sì. Le informazioni sulla vulnerabilità sono state divulgate pubblicamente. Questa vulnerabilità è stata identificata con il codice CAN-2005-2126.

**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco, ma era a conoscenza dell'esistenza di esempi di codice di prova pubblicati.

**L'applicazione di questo aggiornamento per la protezione consente di proteggere gli utenti da eventuale codice di prova pubblicato che tenti di sfruttare la vulnerabilità?**
Sì. L'aggiornamento per la protezione affronta la vulnerabilità per cui è stato pubblicato codice di prova.
