Bollettino Microsoft sulla sicurezza MS13-098 - Critico
=======================================================

Una vulnerabilità in Windows può consentire l'esecuzione di codice in modalità remota (2893294)
-----------------------------------------------------------------------------------------------

Data di pubblicazione: 10 dicembre 2013 | Data di aggiornamento: 29 luglio 2014

**Versione:** 1.6

Informazioni generali
---------------------

### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità di Microsoft Windows che è stata segnalata privatamente. La vulnerabilità può consentire l'esecuzione di codice in modalità remota se un utente o un'applicazione esegue o installa un file firmato PE (Portable Executable) appositamente predisposto in un sistema interessato.

Questo aggiornamento per la protezione è considerato di livello critico per tutte le versioni supportate di Windows. Per ulteriori informazioni, vedere la sezione **Software interessato e Software non interessato**.

L'aggiornamento per la protezione risolve la vulnerabilità modificando il modo in cui la funzione WinVerifyTrust gestisce la verifica della firma Authenticode di Windows per i file PE (Portable Executable). Per ulteriori informazioni sulla vulnerabilità, cercare la voce relativa alla vulnerabilità specifica nella sottosezione **Domande frequenti** più avanti in questo bollettino.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](https://support.microsoft.com/kb/294871).

Per gli amministratori e le installazioni delle organizzazioni o gli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare immediatamente l'aggiornamento utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità degli aggiornamenti tramite il servizio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

Domande frequenti sull'aggiornamento
------------------------------------

<span id="sectionToggle1"></span>
**Perché questo bollettino è stato rivisto il 29 luglio 2014? 
**In precedenza, era stato specificato nel presente bollettino che Microsoft avrebbe rilasciato, come funzionalità predefinita, il comportamento di verifica restrittiva della firma Authenticode descritto nell'[Advisory Microsoft sulla sicurezza 2915720](http://technet.microsoft.com/en-us/security/advisory/2915720). Tuttavia, in seguito alle attività per l'adattamento di tale modifica condotte con i clienti, è emerso che l'impatto sul software esistente sarebbe stato elevato. Pertanto, Microsoft ha deciso di non applicare più tale comportamento come requisito predefinito. Le funzionalità correlate alla verifica restrittiva della firma rimangono tuttavia valide e possono essere attivate a discrezione dell'utente. Vedere l'[Advisory Microsoft sulla sicurezza 2915720](http://technet.microsoft.com/en-us/security/advisory/2915720) per ulteriori informazioni.

**L'aggiornamento include modifiche di funzionalità relative alla sicurezza?** 
Sì. Oltre alle modifiche elencate nella sezione Informazioni sulle vulnerabilità del presente bollettino, questo aggiornamento contiene modifiche a un comportamento predefinito della verifica della firma Authenticode di Windows che verrà attivato solo dietro consenso esplicito dell'utente. Tenere presente che questa modifica non viene attivata per impostazione predefinita con l'installazione dell'aggiornamento. Per ulteriori informazioni e istruzioni su come attivare la modifica, vedere l'[Advisory Microsoft sulla sicurezza 2915720](http://technet.microsoft.com/en-us/security/advisory/2915720).

**Windows 8.1 Preview e Windows Server 2012 R2 Preview sono interessati da una delle vulnerabilità risolte in questo bollettino?** 
Sì. L'aggiornamento 2893294 è disponibile per Windows 8.1 Preview e Windows Server 2012 R2 Preview. Ai clienti che utilizzano questi sistemi operativi si consiglia di applicare l'aggiornamento ai loro sistemi. L'aggiornamento è disponibile solo su [Windows Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

**Dopo aver applicato questo aggiornamento la firma digitale del software in uso non è valida; come bisogna comportarsi? **
Questo aggiornamento causa l'esecuzione di un controllo severo della firma Authenticode di Windows da parte di WinVerifyTrust per i file PE. Dopo aver applicato l'aggiornamento, i file PE vengono considerati "non firmati" se Windows identifica in essi contenuto non conforme alla specifica Authenticode. Questo può avere un impatto su alcuni programmi di installazione. Se si sta utilizzando un programma di installazione interessato dal problema, Microsoft consiglia di utilizzare un programma di installazione che estragga il contenuto solo dalle parti convalidate del file firmato.

**Se si utilizza una versione precedente del software discusso nel presente bollettino, come bisogna comportarsi?** 
I prodotti software elencati in questo bollettino sono stati sottoposti a test per determinare quali versioni sono interessate dalla vulnerabilità. Le altre versioni sono al termine del ciclo di vita del supporto. Per ulteriori informazioni sul ciclo di vita dei prodotti, visitare il sito Web [Ciclo di vita del supporto Microsoft](http://support.microsoft.com/common/international.aspx?rdpath=gp;%5Bln%5D;lifecycle).

Per evitare di esporre i propri sistemi a potenziali vulnerabilità, si raccomanda ai clienti che utilizzano versioni precedenti del software di eseguire quanto prima la migrazione a versioni supportate. Per informazioni sulla disponibilità del supporto per la versione del software in uso, vedere [Selezionare un prodotto per informazioni sul ciclo di vita](http://support.microsoft.com/gp/lifeselect). Per ulteriori informazioni sui service pack relativi a queste versioni del software, vedere [Criteri relativi al ciclo di vita del supporto per i service pack](http://support.microsoft.com/gp/lifesupsps).

I clienti che hanno ancora la necessità di servizi di supporto per versioni precedenti del software sono invitati a contattare il loro rappresentante del team Microsoft, il responsabile tecnico dei rapporti con i clienti oppure il rappresentante del partner Microsoft appropriato per informazioni sui servizi di supporto personalizzati. Quando si effettua la chiamata, richiedere di parlare con il responsabile locale delle vendite per i servizi di supporto Premier. Vulnerabilità legata alla convalida della firma mediante WinVerifyTrust - CVE-2013-3900
---------------------------------------------------------------------------------------

<span id="sectionToggle3"></span>
È stata riscontrata una vulnerabilità legata all'esecuzione di codice in modalità remota nel modo in cui la funzione WinVerifyTrust gestisce la verifica della firma Authenticode di Windows per i file PE (Portable Executable). Un utente malintenzionato anonimo può sfruttare la vulnerabilità modificando un file eseguibile firmato esistente per sfruttare parti non verificate del file in modo da aggiungere codice dannoso al file senza annullare la convalida della firma. Sfruttando questa vulnerabilità, un utente malintenzionato può assumere il pieno controllo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2013-3900](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3900).

### Fattori attenuanti

Microsoft non ha individuato alcun fattore attenuante per questa vulnerabilità.

### Soluzioni alternative

Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.

### Domande frequenti

**Qual è la portata o l'impatto di questa vulnerabilità?** 
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota.

**Quali sono le cause di questa vulnerabilità?** 
La vulnerabilità si presenta quando la funzione WinVerifyTrust convalida erroneamente il digest di un file PE appositamente predisposto durante la verifica della firma Authenticode di Windows.

**Che cos'è la verifica della firma Authenticode di Windows? 
**La verifica della firma Authenticode di Windows comprende due attività principali: controllo della firma su oggetti specificati e verifica dell'attendibilità. Queste attività vengono effettuate dalla funzione WinVerifyTrust, che esegue un controllo della firma per poi trasferire la domanda a un provider di attendibilità che supporta l'identificatore dell'azione, se esistente. Per informazioni più tecniche riguardanti la funzione WinVerifyTrust, vedere [Funzione WinVerifyTrust](http://msdn.microsoft.com/en-us/library/aa388208(vs.85).aspx).

Per un'introduzione ad Authenticode, vedere [Introduzione alla firma codice](http://msdn.microsoft.com/en-us/library/ms537361(v=vs.85).aspx).

**Che cos'è il formato di firma Authenticode PE (Portable Executable) di Windows?** 
Authenticode è un formato di firma digitale utilizzato per definire l'origine e l'integrità di file di software binari. Authenticode utilizza i dati firmati PKCS (Public-Key Cryptography Standard) 7 e i certificati X.509 per associare un file binario con firma Authenticode all'identità di un autore di software. Il termine "Authenticode" relativo alla firma fa riferimento a un formato di firma digitale generato e verificato tramite la funzione di verifica della firma Authenticode. Per ulteriori informazioni su questo formato, vedere [Formato di firma Authenticode PE (Portable Executable) di Windows](http://www.microsoft.com/whdc/winlogo/drvsign/authenticode_pe.mspx).

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?** 
Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato che sfrutti questa vulnerabilità può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

**In che modo un utente malintenzionato può sfruttare questa vulnerabilità?** 
Per sfruttare questa vulnerabilità, è necessario che un utente o un'applicazione eseguano o installino un file PE firmato appositamente predisposto. Un utente malintenzionato potrebbe modificare un file firmato esistente per includere codice dannoso senza annullare la convalida della firma. Questo codice viene eseguito nel contesto del privilegio in cui è stato avviato il file PE firmato.

In uno scenario di attacco tramite posta elettronica, un utente malintenzionato può sfruttare la vulnerabilità inviando a un utente un messaggio di posta elettronica contenente il file PE appositamente predisposto e convincendolo ad aprire tale file.

In uno scenario di attacco dal Web, un utente malintenzionato deve ospitare un sito Web contenente un file PE appositamente predisposto. Inoltre, i siti Web manomessi e quelli che accettano o ospitano contenuti forniti dagli utenti possono presentare contenuti appositamente predisposti che possono essere utilizzati per sfruttare questa vulnerabilità. Non è in alcun modo possibile obbligare un utente a visitare un sito Web che ospita il file PE appositamente predisposto. L'utente malintenzionato dovrebbe invece convincere gli utenti a visitare il sito Web, in genere inducendoli a fare clic su un collegamento in un messaggio di posta elettronica o di Instant Messenger che li indirizzi al sito.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?** 
I sistemi maggiormente a rischio sono workstation e server terminal.

**Quali sono gli scopi dell'aggiornamento?** 
L'aggiornamento risolve la vulnerabilità modificando il modo in cui la funzione WinVerifyTrust gestisce la verifica della firma Authenticode di Windows per i file PE (Portable Executable).

**Nota** Questo aggiornamento causa l'esecuzione di un controllo severo della firma Authenticode di Windows da parte di WinVerifyTrust per i file PE. Dopo aver applicato l'aggiornamento, i file PE vengono considerati "non firmati" se Windows identifica in essi contenuto non conforme alla specifica Authenticode. Questo può avere un impatto su alcuni programmi di installazione. Se si sta utilizzando un programma di installazione interessato dal problema, Microsoft consiglia di utilizzare un programma di installazione che estragga il contenuto solo dalle parti convalidate del file firmato.

**Al momento del rilascio di questo bollettino, le informazioni sulla vulnerabilità erano disponibili pubblicamente?** 
No. Microsoft ha ricevuto informazioni sulla vulnerabilità grazie a un'indagine coordinata.

**Al momento del rilascio di questo bollettino, erano già stati segnalati a Microsoft attacchi basati sullo sfruttamento di questa vulnerabilità?** 
Sì. Microsoft è a conoscenza di attacchi mirati che tentano di sfruttare questa vulnerabilità.
