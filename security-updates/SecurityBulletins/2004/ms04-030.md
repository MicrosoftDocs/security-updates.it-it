Security Bulletin

Bollettino Microsoft sulla sicurezza MS04-030 - Importante
==========================================================

Titolo del bollettino: una vulnerabilità del gestore dei messaggi XML di WebDAV può causare la negazione del servizio (824151)
------------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 12 ottobre 2004

**Data di pubblicazione:** 12 ottobre 2004
**Versione:** 1.0

#### Riepilogo

**Destinatari:** clienti che utilizzano Microsoft® Windows®

**Effetti della vulnerabilità:** negazione del servizio

**Livello di gravità massimo:** importante.

**Raccomandazioni:** si raccomanda di installare l'aggiornamento il più presto possibile.

**Aggiornamenti per la protezione sostituiti:** nessuno.

**Avvertenze:** questo aggiornamento include una modifica della funzionalità che potrebbe interessare alcune applicazioni che trasmettono richieste WebDAV valide caratterizzate da numerosi attributi XML. Per ulteriori informazioni, consultare "L'aggiornamento include anche altre modifiche alle funzionalità?" nella sezione Domande frequenti correlate a questo aggiornamento per la protezione.

**Software verificato e posizioni per il download dell'aggiornamento per la protezione:**

**Software interessato:**

-   Microsoft Windows XP e Microsoft Windows XP Service Pack 1 – Download dell'aggiornamento
-   Microsoft Windows XP 64-Bit Edition Service Pack 1 – Download dell'aggiornamento
-   Microsoft Windows XP 64-Bit Edition versione 2003 – Download dell'aggiornamento
-   Microsoft Windows Server™ 2003 – Download dell'aggiornamento
-   Microsoft Windows Server 2003 64-Bit Edition – Download dell'aggiornamento

**Software non interessato:**

-   Microsoft Windows NT Server 4.0 Service Pack
-   Microsoft Windows NT 4.0 Server Terminal Server Edition Service Pack 6
-   Microsoft Windows 98, Microsoft Windows 98 Second Edition (SE) e Microsoft Windows Millennium Edition (ME)

**Componenti interessati:**

**Componenti non interessati:**

-   Internet Information Server 4.0

I prodotti software inclusi nell'elenco sono stati sottoposti a test per determinare se tali versioni sono interessate dalla vulnerabilità. Domande frequenti correlate a questo aggiornamento per la protezione
--------------------------------------------------------------------

<span></span>
**L'aggiornamento include anche altre modifiche alle funzionalità?**
Sì. Oltre alle modifiche elencate nella sezione Dettagli delle vulnerabilità del presente bollettino, l'aggiornamento richiede l'applicazione di nuovi limiti ai documenti XML accettati da WebDAV. A causa di questa modifica, tutte le richieste valide che superano questi limiti saranno restituite al client come richieste non riuscite. È possibile configurare questi nuovi limiti utilizzando le voci della metabase. Se un'applicazione richiede documenti che sarebbero rifiutati in ragione di questi nuovi limiti, è possibile regolare le voci della metabase per assicurare il supporto del contenuto richiesto. Le nuove configurazioni della metabase sono descritte nell'articolo della Microsoft Knowledge Base [824151](http://support.microsoft.com/?kbid=824151).

Dettagli della vulnerabilità
----------------------------

<span></span>
#### Vulnerabilità WebDAV - CAN-2004-0718:

Esiste una vulnerabilità legata alla possibilità di [negazione del servizio](http://go.microsoft.com/fwlink/?linkid=21142x) che potrebbe consentire a un pirata informatico di inviare una richiesta WebDAV appositamente predisposta a un server che esegue IIS e WebDAV. Un pirata informatico potrebbe sfruttare la vulnerabilità e far consumare a WebDAV tutta la memoria e il tempo della CPU disponibili su un server interessato. Per il ripristino del normale funzionamento sarebbe quindi necessario riavviare il servizio IIS.

#### Fattori attenuanti della vulnerabilità WebDAV - CAN-2004-0718:

-   Questa vulnerabilità può essere sfruttata in remoto da un pirata informatico solo se questo riesce a stabilire una sessione Web con un server interessato.
-   Per impostazione predefinita, in Windows XP e Windows Server 2003, ad eccezione di Windows Server 2003 Web Server Edition, IIS non viene installato assieme al sistema operativo.
-   IIS 5.0, integrato in Windows 2000, è l'unica versione in cui WebDAV è abilitato per impostazione predefinita.
-   Windows NT 4.0 non è interessato dalla vulnerabilità.

#### Soluzioni alternative per la vulnerabilità WebDAV - CAN-2004-0718:

Microsoft ha verificato le soluzioni alternative seguenti. Sebbene queste soluzioni alternative non eliminino la vulnerabilità sottostante, consentono di bloccare gli attacchi noti. Se una soluzione alternativa riduce la funzionalità, viene identificata di seguito.

-   Disabilitare WebDAV in IIS 5.0 (se non è necessario).

#### Domande frequenti relative alla vulnerabilità WebDAV - CAN-2004-0718:

**Qual è la portata o l'impatto di questa vulnerabilità?** Si tratta di una vulnerabilità di [negazione del servizio](http://go.microsoft.com/fwlink/?linkid=21142) Un pirata informatico potrebbe sfruttare la vulnerabilità e far consumare a WebDAV tutta la memoria e il tempo della CPU disponibili su un server interessato. Per il ripristino del normale funzionamento sarebbe quindi necessario riavviare il servizio IIS.

**Quali sono le cause di questa vulnerabilità?**
WebDAV non prevede alcun limite al numero di attributi che possono essere specificati per ciascun elemento XML.

**Per quale motivo questa vulnerabilità richiede l'aggiornamento a MSXML 3.0 Service Pack 5, incluso in questo aggiornamento per la protezione?**
Microsoft XML Parser è un componente fondamentale del sistema operativo, utilizzato dalle applicazioni per elaborare i documenti XML. Questo componente è utilizzato da WebDAV e da applicazioni di terzi. Questo aggiornamento è necessario per consentire a WebDAV di limitare il numero di attributi dei documenti inviati di cui tenta l'elaborazione.

**È necessario installare MSXML 3.0 Service Pack 5 separatamente?**
No. Il pacchetto dell'aggiornamento include sia l'aggiornamento per WebDAV che i file di aggiornamento a MSXML 3.0 Service Pack 5.

**Che cos'è WebDAV?**
WebDAV è un'estensione dello standard della specifica HTTP. DAV è l'acronimo di "Distributed Authoring And Versioning". WebDAV consente agli utenti autorizzati di aggiungere e gestire i contenuti in remoto su un server Web. Per impostazione predefinita, WebDAV è abilitato quando IIS è abilitato su Windows 2000. Per impostazione predefinita, WebDAV non viene installato su IIS 5.1 o IIS 6.0.

**Qual è il problema di gestione delle richieste HTTP da parte di WebDAV?**
Prima dell'ultimo aggiornamento del parser XML, MSXML 3.0, né WebDAV né altre applicazioni di terzi che si avvalevano di Microsoft XML Parser potevano limitare il numero di attributi che Microsoft XML Parser tentava di elaborare nei documenti inviati.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**
Si tratta di una vulnerabilità di negazione del servizio. Pertanto, un eventuale aggressore potrebbe causare un'interruzione dei normali servizi. Con il riavvio del servizio interessato, il server torna alla completa funzionalità. Tuttavia, il servizio rimane comunque soggetto ad attacchi di questo tipo finché non viene applicato l'aggiornamento.

**Quali capacità sono necessarie per sfruttare questa vulnerabilità?**
Un pirata informatico in grado di inviare una richiesta WebDAV a un server interessato da questa vulnerabilità può tentare di sfruttarla. Poiché le richieste WebDAV sono trasmesse attraverso la medesima porta di quelle HTTP (in genere la porta 80), un pirata informatico in grado di stabilire una connessione con un server Web interessato dalla vulnerabilità può tentare di sfruttarla. È possibile accedere al componente interessato anche con altri metodi, utilizzando ad esempio un'applicazione che riceva richieste HTTP e utilizzi Microsoft XML Parser per elaborare i documenti XML o tramite un'applicazione simile a WebDAV che elabori i documenti XML utilizzando il componente vulnerabile in modalità locale o remota.

**In quale modo un pirata informatico può sfruttare questa vulnerabilità?**
Per sfruttare questa vulnerabilità, un pirata informatico deve inviare al sito Web interessato un messaggio HTTP volto a incrementare fino al 100% l'utilizzo della CPU nel server IIS durante l'elaborazione del messaggio da parte di IIS. Maggiore il numero di attributi XML specificati per ciascun elemento XML nel messaggio XML, più tempo impiegherà il server IIS a elaborare il messaggio XML. Ciò potrebbe provocare una condizione di negazione del servizio nel corso dell'elaborazione dei messaggi da parte di IIS. In questo caso, per ripristinare il normale funzionamento del server sarà necessario riavviare il servizio.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**
I sistemi principalmente a rischio sono i server che eseguono sia i servizi IIS che WebDAV.

**È possibile sfruttare questa vulnerabilità tramite Internet?**
Sì. Un pirata informatico può sfruttare questa vulnerabilità tramite Internet. Le configurazioni predefinite standard dei firewall e le procedure consigliate per la configurazione dei firewall consentono di proteggersi dagli attacchi provenienti da Internet. Microsoft fornisce informazioni su come è possibile proteggere il proprio PC. [Informazioni per gli utenti finali: visitare il sito Web](http://www.microsoft.com/italy/athome/security/default.mspx)Protezione dei PC. Informazioni per i professionisti IT: visitare il [sito Web Security Guidance Center](http://www.microsoft.com/italy/security/guidance/).

**Quali sono gli scopi dell'aggiornamento?**
L'aggiornamento elimina la vulnerabilità facendo sì che WebDAV utilizzi le nuove proprietà del parser XML per limitare il numero di attributi XML specificati per ciascun elemento nei documenti XML accettati nel caso di richieste basate su WebDAV.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private. Secondo le informazioni in possesso di Microsoft, al momento della pubblicazione di questo bollettino l'esistenza di questa vulnerabilità non era stata ancora divulgata pubblicamente.
