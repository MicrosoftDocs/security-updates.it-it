Security Bulletin

Bollettino Microsoft sulla sicurezza MS10-031 - Critico
=======================================================

Una vulnerabilità in Microsoft Visual Basic, Applications Edition può consentire l'esecuzione di codice in modalità remota (978213)
-----------------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 11 maggio 2010 | Aggiornamento: mercoledì 19 maggio 2010

**Versione:** 1.1

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità segnalata privatamente in Microsoft Visual Basic, Applications Edition. La vulnerabilità può consentire l'esecuzione di codice in modalità remota se un'applicazione host apre e inoltra un file appositamente predisposto a Visual Basic, Applications Edition. Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato che sfrutti questa vulnerabilità può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

Questo aggiornamento per la protezione è considerato di livello critico per tutte le versioni supportate di Microsoft Visual Basic for Applications SDK e le applicazioni di terze parti che utilizzano Microsoft Visual Basic, Applications Edition. Questo aggiornamento per la protezione è inoltre considerato di livello importante per tutte le edizioni supportate di Microsoft Office XP, Microsoft Office 2003 e Microsoft Office System 2007. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

L'aggiornamento risolve la vulnerabilità modificando il modo in cui Visual Basic, Applications Edition cerca controlli ActiveX incorporati nei documenti. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Microsoft consiglia di applicare l'aggiornamento immediatamente.

**Problemi noti.** L'[articolo della Microsoft Knowledge Base 978213](http://support.microsoft.com/kb/978213) descrive i problemi attualmente conosciuti che gli utenti potrebbero riscontrare durante l'installazione di questo aggiornamento per la protezione. L'articolo illustra inoltre le soluzioni consigliate in grado di risolvere questi problemi. Quando i problemi noti e le soluzioni consigliate si riferiscono solo a determinate versioni di questo software, nell'articolo vengono forniti collegamenti ad altri articoli.

Domande frequenti sull'aggiornamento per la protezione  
------------------------------------------------------
  
<span></span>
**Perché Microsoft ha rivisto il bollettino in data 19 maggio 2010?**    
Microsoft ha rivisto questo bollettino per chiarire che gli aggiornamenti per Microsoft Office interessano tutte le applicazioni Microsoft Office supportate e altro software Microsoft Office contenente una versione vulnerabile di VBE6.dll. Questo include, ma non solo, le versioni supportate di Microsoft Office Visio e Microsoft Office Project. È incluso inoltre il software Office che contiene VBE6.dll ma non accede al codice vulnerabile come le versioni supportate di Microsoft Office Outlook e Microsoft Office OneNote.
  
Questa modifica riguarda esclusivamente il bollettino. Non sono previste modifiche alla logica di rilevamento o ai file di aggiornamento. I clienti che hanno già aggiornato i propri sistemi, inclusi i clienti che hanno attivato l'aggiornamento automatico, non devono eseguire ulteriori operazioni. È possibile che i clienti che non hanno già installato questo aggiornamento debbano rivalutare la necessità di installarlo nei propri sistemi in base alle informazioni contenute nella tabella Software interessato modificata e alle informazioni fornite nella presente Domanda frequente sull'aggiornamento.
  
**L'aggiornamento è relativo a un componente condiviso utilizzato da Microsoft Office. Verrà offerto al software Microsoft Office che contiene il componente condiviso, anche se tale software non accede al codice vulnerabile?**    
Sì, l'aggiornamento verrà offerto ai sistemi in cui viene rilevato il componente Office vulnerabile condiviso. Ciò vale anche per i casi in cui il software Office contiene VBE6.dll ma non accede al codice vulnerabile. Consultare la tabella seguente per gli esempi di software Office a cui si applica questo aggiornamento.
  
| Microsoft Office                                                                            | VBE6.dll è incluso in questo software? | Il software accede al codice vulnerabile?     |  
|---------------------------------------------------------------------------------------------|----------------------------------------|-----------------------------------------------|  
| Microsoft Office XP, Microsoft Office 2003, Microsoft Office System 2007                    | No                                     | Sì, componenti quali Word, Excel e PowerPoint |  
| Microsoft Office OneNote 2003, Microsoft Office OneNote 2007                                | No                                     | No                                            |  
| Microsoft Office Outlook 2003, Microsoft Office Outlook 2007                                | No                                     | No                                            |  
| Microsoft Office Project 2002, Microsoft Office Project 2003, Microsoft Office Project 2007 | No                                     | No                                            |  
| Microsoft Office Visio 2002, Microsoft Office Visio 2003, Microsoft Office Visio 2007       | No                                     | No                                            |
  
**Nota** Utilizzare questa tabella solo come riferimento. Contiene un campione delle domande più frequenti relative al software.
  
Nei casi in cui l'aggiornamento viene offerto, Microsoft consiglia ai clienti di installarlo indipendentemente dal fatto che il software acceda o meno al codice vulnerabile. L'installazione dell'aggiornamento assicura che la versione del file VBE6.dll utilizzata dal software Microsoft Office sia quella più aggiornata.
  
**L'offerta di aggiornamento a una versione del software non esposta a vulnerabilità costituisce un problema nel meccanismo di aggiornamento Microsoft?**    
No, il meccanismo di aggiornamento funziona correttamente poiché è in grado di rilevare una versione dei file del sistema precedente rispetto a quella del pacchetto di aggiornamento e quindi di fornire l'aggiornamento necessario.
  
**Dove si trova la sezione Informazioni sui file?**    
Fare riferimento alle tabelle contenute nella sezione **Distribuzione dell'aggiornamento per la protezione** per individuare la sezione Informazioni sui file.
  
**Sono stati applicati tutti gli aggiornamenti per la protezione Microsoft richiesti, tuttavia è ancora presente nel sistema una versione interessata del runtime di Visual Basic, Applications Edition (VBE6.dll). Come aggiornare questa DLL?**    
In alcuni casi, nel sistema potrebbe essere ancora presente una versione interessata di VBE6.dll anche dopo l'installazione degli aggiornamenti per la protezione richiesti per Microsoft Office e dell'aggiornamento per Microsoft Visual Basic, Applications Edition specificato in questo bollettino.
  
Se VBE6.dll è stato installato nel sistema da una versione di Microsoft Office supportata, l'applicazione dell'aggiornamento per la protezione per la versione interessata di Microsoft Office sostituirà VBE6.dll con la versione aggiornata, che risolve la vulnerabilità descritta in questo bollettino. Tuttavia, se VBE6.dll è stato installato nel sistema da un'applicazione di terze parti, potrebbe essere necessario installare un aggiornamento per tale programma.
  
Per aggiornare VBE6.dll per applicazioni di terze parti, sono disponibili due metodi, a seconda dall'implementazione di VBA dell'applicazione di terze parti. Se si è certi che l'applicazione di terze parti sia conforme alle procedure consigliate per l'utilizzo di un componente condiviso, ad esempio un assembly side-by-side, l'applicazione dell'aggiornamento di Microsoft Visual Basic, Applications Edition (KB974945) sostituirà VBE6.dll nella posizione condivisa con la versione aggiornata, che risolve la vulnerabilità descritta in questo bollettino.
  
Se invece l'applicazione di terze parti non inserisce VBE6.dll nella posizione condivisa in base alle procedure consigliate, è necessario contattare lo sviluppatore dell'applicazione di terze parti per ottenere una versione aggiornata dell'applicazione, contenente una versione più recente di VBE6.dll che risolva la vulnerabilità descritta in questo bollettino. Tenere presente che non tutte le applicazioni di terze parti che installano VBE6.dll sono esposte alla vulnerabilità descritta in questo bollettino, in quanto l'applicazione potrebbe non utilizzare VBE6.dll per ricercare i controlli ActiveX in modo sfruttabile. Tuttavia, soltanto lo sviluppatore dell'applicazione è in grado di determinare se la vulnerabilità esiste.
  
**Questo aggiornamento per la protezione si applica solo al software Microsoft. Come determinare se applicazioni di terze parti hanno distribuito una versione interessata del runtime di Visual Basic, Applications Edition (VBE6.dll) nel sistema?**    
Le applicazioni di terze parti che supportano VBA potrebbero distribuire VBE6.dll in una posizione non aggiornata da questo aggiornamento per la protezione. Se si dispone di un'applicazione di terze parti che include la propria copia di VBE6.dll, per essere certi che il sistema in uso sia protetto completamente dalla vulnerabilità descritta in questo bollettino è consigliabile contattare direttamente lo sviluppatore o il fornitore responsabile del supporto per l'applicazione di terze parti.
  
Per ulteriori informazioni su come eseguire l'analisi delle copie di VBE6.dll nel sistema, vedere l'[articolo della Microsoft Knowledge Base 978213](http://support.microsoft.com/kb/978213). L'analisi di tutte le copie di VBE6.dll presenti nel sistema può aiutare a identificare copie potenzialmente vulnerabili che potrebbero essere state installate da applicazioni di terze parti.
  
**Sono uno sviluppatore di applicazioni di terze parti e utilizzo il runtime di Microsoft Visual Basic, Applications Edition nella mia applicazione. L'applicazione è vulnerabile e come è possibile aggiornarla?**    
Gli sviluppatori che ridistribuiscono il runtime di Microsoft Visual Basic, Applications Edition (VBE6.dll) devono aggiornare la versione del runtime di Microsoft Visual Basic, Applications Edition installata con l'applicazione in uso scaricando una versione aggiornata di Microsoft Visual Basic for Applications SDK dal sito [Summit Software Company](http://www.summsoft.com/). Per ulteriori informazioni sulle procedure consigliate in merito all'utilizzo di componenti ridistribuiti, vedere l'[articolo della Microsoft Knowledge Base 835322](http://support.microsoft.com/kb/835322) nonché l'articolo di MSDN [Applicazioni isolate e assembly side-by-side](http://msdn.microsoft.com/it-it/library/aa375193.aspx).
  
**Sono un ISV. Dove è disponibile l'aggiornamento per Microsoft Visual Basic for Applications SDK?**    
L'aggiornamento è disponibile sul sito [Summit Software Company](http://www.summsoft.com/). Summit Software Company è un fornitore globale di prodotti software per la personalizzazione delle applicazioni e servizi di supporto dell'integrazione per i fornitori di software indipendenti (ISV) e gli sviluppatori aziendali. Nel giugno 1996 Summit Software ha stretto un accordo con Microsoft per vendere Microsoft Visual Basic, Applications Edition nonché la tecnologia e i servizi a valore aggiunto correlati. Summit continua a vendere e a supportare Microsoft VBA.
  
**Perché questo aggiornamento è di livello critico per Microsoft Visual Basic, Applications Edition e solo di livello importante per Microsoft Office?**    
Microsoft Office XP e versioni successive dispongono di una funzionalità predefinita che richiede all'utente di aprire, salvare o annullare un documento prima di aprirlo. Questo fattore attenuante riduce la vulnerabilità a un livello importante perché richiede più azioni dell'utente per sfruttarla.
  
**L'utente utilizza Microsoft Office 2007 Service Pack 1. Sono state incluse funzionalità di protezione aggiuntive in questo aggiornamento?**    
Sì, il modello di servizio di Microsoft Office 2007 prevede che, quando gli utenti di Microsoft Office 2007 Service Pack 1 installano il presente aggiornamento, il loro sistema venga aggiornato con le funzionalità inizialmente rilasciate con Microsoft Office 2007 Service Pack 2. Tutti gli aggiornamenti rilasciati dopo il 24 aprile 2009 per Microsoft Office 2007 includono tali funzionalità di protezione, introdotte in Microsoft Office 2007 Service Pack 2. Anche se il presente aggiornamento è stato verificato in modo completo, si consiglia, come per tutti gli aggiornamenti, di eseguire i test adeguati all'ambiente e alla configurazione del sistema in uso.

Vulnerabilità legata al danneggiamento della memoria stack in VBE6.dll - CVE-2010-0815
--------------------------------------------------------------------------------------

<span></span>
Esiste una vulnerabilità legata all'esecuzione di codice in modalità remota relativa al modo in cui Microsoft Visual Basic, Applications Edition cerca i controlli ActiveX. Questa vulnerabilità può consentire l'esecuzione di codice in modalità remota se un'applicazione host apre e inoltra un file appositamente predisposto al runtime di Visual Basic, Applications Edition. Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato che sfrutti questa vulnerabilità può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2010-0815](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0815).

#### Fattori attenuanti della vulnerabilità legata al danneggiamento della memoria stack in VBE6.dll - CVE-2010-0815

Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:

-   Per impostazione predefinita, Microsoft Office System 2007 visualizza un avviso di protezione prima di attivare un controllo ActiveX incorporato in un documento di Office. Gli utenti che scelgono di non attivare il controllo sono protetti da questa impostazione predefinita. Per ulteriori informazioni, vedere l'articolo di Microsoft Office Online [Attivazione o disattivazione dei controlli ActiveX nei documenti di Office](http://office.microsoft.com/it-it/help/ha100310671040.aspx).
-   Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere gli stessi diritti utente dell'utente locale. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.
-   In uno scenario di attacco basato sul Web, l'utente malintenzionato potrebbe pubblicare un sito Web contenente una pagina utilizzata per sfruttare la vulnerabilità. #### Domande frequenti sulla vulnerabilità legata al danneggiamento della memoria stack in VBE6.dll - CVE-2010-0815

**Qual è la portata o l'impatto di questa vulnerabilità?**  
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota in Microsoft Visual Basic, Applications Edition (VBA). Sfruttando questa vulnerabilità, un utente malintenzionato può assumere il pieno controllo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

**Quali sono le cause di questa vulnerabilità?**  
La vulnerabilità è causata dal modo in cui Visual Basic, Applications Edition cerca controlli ActiveX in un documento che supporta VBA. Di conseguenza, è possibile che un'applicazione host, quale Microsoft Office o un'applicazione di terze parti sviluppata per la programmabilità di Visual Basic, inoltri al runtime di VBA un documento appositamente predisposto con controlli ActiveX incorporati, creando una condizione che può consentire l'esecuzione di codice non autorizzato.

**Che cos'è Visual Basic, Applications Edition (VBA)?**  
Microsoft VBA è una tecnologia di sviluppo che consente di realizzare pacchetti di applicazioni client desktop e di integrarli con dati e sistemi esistenti. Microsoft VBA si basa sul sistema di sviluppo Microsoft Visual Basic ed è incluso nei prodotti Microsoft Office, che utilizzano tale tecnologia per l'esecuzione di determinate funzioni. È possibile utilizzare VBA anche per creare applicazioni personalizzate basate su un'applicazione host esistente.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**  
Sfruttando questa vulnerabilità, un utente malintenzionato può assumere il pieno controllo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**  
Affinché un attacco riesca, è necessario che un utente apra un documento appositamente predisposto che supporti VBA. Tale documento può essere un qualsiasi tipo di documento che supporti VBA, ad esempio un documento di Word, un foglio di lavoro di Excel o una presentazione di PowerPoint.

In uno scenario di attacco tramite posta elettronica, l'utente malintenzionato potrebbe sfruttare la vulnerabilità inviando un file Office appositamente predisposto e inducendo l'utente ad aprirlo.

In uno scenario di attacco dal Web, l'utente malintenzionato deve pubblicare un sito Web contenente un documento che supporti VBA, ad esempio un file di Office utilizzato per tentare di sfruttare questa vulnerabilità. Inoltre, i siti Web manomessi e quelli che accettano o ospitano contenuti forniti dagli utenti potrebbero presentare contenuti appositamente predisposti per sfruttare questa vulnerabilità. Poiché non è in alcun modo possibile obbligare gli utenti a visitare un sito Web appositamente predisposto, Un utente malintenzionato, invece, deve convincere gli utenti a visitare il sito Web, in genere inducendoli a fare clic su un collegamento contenuto in un messaggio di posta elettronica o di Instant Messenger che li indirizzi al sito Web dell'utente malintenzionato. Successivamente, deve convincerli ad aprire il file Office appositamente predisposto.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**  
I sistemi maggiormente a rischio sono le workstation e i server terminal che eseguono Microsoft Visual Basic, Applications Edition o Microsoft Office. I server potrebbero essere maggiormente a rischio se gli amministratori consentono agli utenti di accedere ai server ed eseguire programmi. Tuttavia, le procedure consigliate scoraggiano fortemente questa attribuzione di privilegi.

**Quali sono gli scopi dell'aggiornamento?**  
L'aggiornamento risolve la vulnerabilità modificando il modo in cui Microsoft Visual Basic, Applications Edition cerca controlli ActiveX incorporati nei documenti.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**  
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private.

**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**  
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco.
